## 철학
SAZO의 모달은 **최소한의 방해로 필요한 결정을 이끌어낸다** . 모든 모달은 `SazoModal` 공통 프레임을 사용하며, 내부 콘텐츠 구성만 유형별로 달라진다.

- 확인 모달은 `max-w-[400px]`로 컴팩트하게
- 파괴적 액션은 반드시 `color="error"` 버튼으로 경고
- 열기/닫기 모션은 비대칭 이징으로 자연스럽게

## 기본 원칙

<AccordionGroup>

<Accordion title="아나토미 상세">

| 유형 | 상단 패딩 | 좌우 | 하단 (버튼 컨테이너 제외) |
|------|----------|------|----------------------|
| 바텀시트 | 48px | -- | 24px |
| 중앙팝업 (MO) | 48px | 16px | 24px |
| 중앙팝업 (PC) | 48px | 48px | 24px |

PC 중앙팝업: 콘텐츠 높이만큼 최대 높이를 적용한다. 내부 요소 높이가 동적인 경우 고정 높이를 별도 지정할 수 있음 (예: 배송조회 모달 800px). *-- 동적 높이 처리는 미결 사항.*

</Accordion>

<Accordion title="타이틀 생략 규칙">

모달 타이틀(상단 헤더의 제목 텍스트)은 기본적으로 사용한다. 아래 3가지 조건이 **모두** 충족될 때만 생략 가능:
1. 모달 내부에 피쳐드 헤딩 등 **모달 전체에 대응하는 타이틀 역할** 요소가 있을 때 (부분 섹션 타이틀은 해당 안 됨)
2. **유저가 의도적으로 연 모달** 일 것 -- 의도치 않은 타이밍에 뜨는 모달이면 생략하면 안 됨
3. 모달 외부에서 모달의 **성격을 충분히 파악하고 들어올 수 있는 장치** (특히 텍스트)가 있었을 것

</Accordion>

<Accordion title="트랜지션">

콘텐츠에 스크롤이 생기면 바텀 버튼 컨테이너의 **deco(그림자+상단 radius) on** , 스크롤이 사라지면 **off** .

</Accordion>

</AccordionGroup>

## Do / Don't

<Check>

**Do**

- 파괴적 버튼에 `color="error" variant="filled"`
- 취소(좌) + 확인(우) 순서 고정
- 확인 모달은 `max-w-[400px]` 컴팩트
- 확인 모달에는 2개 버튼만
- 제목에 액션 결과 명시 ("삭제하시겠습니까")
- 모달 버튼은 ButtonContainer 사용
- 열기/닫기는 비대칭 이징

</Check>

<Warning>

**Don't**

- 비파괴적 작업에 error 색상 사용
- 좌우 버튼 순서 반대
- 확인 모달에 복잡한 UI 삽입
- 3개 이상 버튼 (선택 과부하)
- 모호한 제목 ("확인")
- 직접 `div + flex`로 레이아웃 구성
- 동일한 이징 사용

</Warning>

## 모달 설계 체크리스트

<Steps>

<Step title="유형 결정">

확인? 폼? 안내? 목록?

</Step>

<Step title="너비 결정">

1~3줄 텍스트 → `400px`, 입력 3~5개 → `576px`, 카드 포함 → 커스텀

</Step>

<Step title="Header">

간단 확인 → 없음, 폼 → 닫기 버튼만, 목록 → 제목 + 닫기

</Step>

<Step title="Footer">

확인만 → Single, 확인/취소 → Double, 폼 내부 버튼 → 없음

</Step>

<Step title="본문 토큰">

제목 `text-xl semibold gray-900`, 설명 `text-base normal gray-700`

</Step>

</Steps>
