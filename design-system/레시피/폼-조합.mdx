## 철학
SAZO의 폼은 **최소한의 인지 부하로 정확한 입력을 유도** 한다. 핵심 원칙은 세 가지다.

- **일관된 간격** : gap-4`(16px)
- **3요소 에러** : 테두리(`border-border-error`) + 아이콘(`LuCircleAlert`) + 메시지(`text-text-error`)를 항상 함께 사용
- **단계적 공개** : 10개 이상 필드는 Progressive Disclosure 패턴으로 분할

## Do / Don't

<Check>

**Do**

- 에러 표시는 테두리 + 아이콘 + 메시지 3가지 조합
- 폼 간격은 `gap-4` (16px) 통일
- 긴 폼은 단계적 공개 패턴으로 분할
- 검증은 `onBlur` 또는 `onSubmit` 모드 사용
- 포커스 시 `border-border-strong` 강한 테두리로 명확한 피드백
- CheckBox variant는 항상 `"filled"`
- Chip 선택은 배경 + 텍스트 동시 변경

</Check>

<Warning>

**Don't**

- 색상으로만 에러 표시 (접근성 문제)
- 필드마다 다른 간격 사용
- 10개 이상 필드를 한 번에 표시
- `onChange` 모드 (매 입력마다 검증, UX 저하)
- 포커스 시각적 피드백 없음
- `"outline"` 체크박스 사용
- 텍스트 색상만 변경

</Warning>

## 폼 설계 시 체크리스트

<Steps>

<Step title="필드 간격">

`gap-4` (16px) 통일

</Step>

<Step title="Label">

`text-label-default` + `text-text-secondary`

</Step>

<Step title="Input radius">

`rounded-xl` (12px) 통일

</Step>

<Step title="에러 3요소">

테두리(`error-300`) + 아이콘(`error-600`) + 메시지(`error-600`)

</Step>

<Step title="비활성">

배경 `bg-surface-disabled` + 텍스트 `text-text-disabled`

</Step>

<Step title="Placeholder">

`text-text-tertiary`

</Step>

<Step title="제출 버튼">

`filled`, `primary`, `lg`, `w-full`, `justify-center`

</Step>

<Step title="약관">

필수는 문구만, 선택은 CheckBox + `border-t` 구분

</Step>

<Step title="안내 문구">

12px + `text-text-secondary`

</Step>

<Step title="폼 하단 여백">

Footer 겹침 방지용 `pb-[7rem]` (스텝 모달)

</Step>

</Steps>
