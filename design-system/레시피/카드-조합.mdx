## 철학
SAZO의 카드는 **정보 밀도와 스캔 가능성의 균형** 을 추구한다. 모든 카드는 동일한 색상 계층(gray-900 → gray-700 → gray-600 → gray-500 → gray-400)을 따르며, 이미지-텍스트 간격과 모서리 반경을 통일하여 어떤 맥락에서든 일관된 시각적 경험을 제공한다.

- 카드 내부의 텍스트 계층은 최대 4단계를 넘지 않는다
- 이미지 크기와 카드 중요도는 비례한다 (48px 알림 → 68px 주문 → 92px 장바구니 → full-width 상품)
- 가격은 항상 `text-text-default` + `600` 이상으로 강조한다

## 카드 형태 변환 규칙
### 패딩-라운드 매칭
카드 내부 패딩과 모서리 라운드를 **동일 값** 으로 매칭한다: 16px 패딩 → 16px 라운드, 24px 패딩 → 24px 라운드.

### PC 카드 → MO 전체 너비 전환
PC에서 카드로 감싸진 콘텐츠가 모바일에서 전체 너비로 전환될 때: 모바일 표준 좌우 패딩(20px) 적용 + **라운드 제거** .

## Do / Don't

<Check>**Do**</Check>

- DTO 변환에 `itemCardParser.fromProductAllResponseDto()` 사용
- 이미지에 `border-blackOpacity-0` 미세 테두리 적용
- 카드 내 상품명은 `500`, 가격은 `600`
- 취소/비활성 상품은 `text-text-disabled` + 이미지 `opacity-50`
- 이미지 크기는 48/68/80/92 기존 스케일에서 선택
- 카드 간 구분은 `<Divider />` 또는 `border-b gray-200`
- 주문 카드에서 상태 아이콘은 `KnownIcon` 사용

<Warning>**Don't**</Warning>

- 수동으로 카드 모델 생성
- 이미지 테두리 없이 배경과 구분 불가 상태로 방치
- 상품명에 `600` 사용 (Title 계층과 혼동)
- 취소 상품을 정상 상품과 동일하게 표시
- 임의의 이미지 크기 사용
- 간격만으로 카드 경계 구분
- 텍스트만으로 상태 표시

## 새 카드 설계 시 체크리스트

<Steps>

<Step title="이미지">

크기를 48/68/80/92 중 선택. `rounded-xl` + `border-blackopacity-0` 적용

</Step>

<Step title="제목">

`text-text-default` + `500` 이상. 줄 제한(`line-clamp-2`) 필수

</Step>

<Step title="부제목">

`text-text-secondary` + `400`. 필요 시 `line-clamp-1`

</Step>

<Step title="가격">

`text-text-default` + `600`. 카드 중요도에 따라 14px ~ 18px

</Step>

<Step title="메타 정보">

`text-text-tertiary` + `400` + 14px 이하

</Step>

<Step title="구분선">

`<Divider />` 또는 `border-b gray-200`

</Step>

<Step title="컨테이너">

최소 `py-4` (16px) 패딩. 테두리가 필요하면 `border gray-200 rounded-lg`

</Step>

<Step title="이미지-텍스트 간격">

`gap-3` (12px) 또는 `gap-4` (16px)

</Step>

<Step title="배지">

컴포넌트 내장 Badge 사용. `xs`/`sm` 크기, `rounded` 모서리

</Step>

<Step title="취소/비활성">

텍스트 `text-text-disabled`, 이미지 `opacity-50`

</Step>

</Steps>
