## 철학
SAZO의 색상 시스템은 핵심 디자인 원칙인 **'정직한 안내자'** 와 **'본질만 남긴 간결함'** 을 시각적으로 구현한다.

- **명확성** : 모든 색상이 기능적 목적을 갖는다. 장식적인 색상 사용을 지양한다.
- **일관성** : 정의된 팔레트와 사용 규칙으로 통일된 시각적 경험을 제공한다.
- **접근성** : 모든 색상 조합은 WCAG AA 레벨 이상의 명도 대비를 충족한다.
- **체계성** : hex 코드가 아닌 역할 기반 **토큰 이름** 으로 관리한다.

<Note>

구체적인 hex 값과 토큰 매핑은 `02 토큰/color/primitive.tokens.json`, `02 토큰/color/semantic.tokens.json`, `02 토큰/css/tokens.css`를 참조한다. 이 문서에서는 hex 값을 직접 기술하지 않는다.

</Note>

## 토큰 아키텍처
<Steps>
  <Step title="Primitive">
    `--gray-900`, `--brand-500` — 색상의 물리적 값. "이 색은 무슨 색인가?"
  </Step>
  <Step title="Semantic (Primitive 참조)">
    `--text-default`, `--surface-error`, `--border-brand` — 색상의 역할. "이 색은 무엇을 의미하는가?"
  </Step>
  <Step title="Tailwind (@theme inline 매핑)">
    `text-text-default`, `bg-surface-error`, `border-border-brand` — `--color-text-*`, `--color-surface-*`, `--color-border-*` 로 매핑된 유틸리티
  </Step>
  <Step title="Component">
    Button bg, Badge text — 유틸리티 클래스로 사용
  </Step>
</Steps>

코드에서는 항상 시맨틱 토큰을 사용한다. Primitive를 직접 사용하지 않는다.

## 네이밍 규칙: Element-first
| 유형 | 패턴 | 설명 |
|------|------|------|
| CSS 변수 | `--{element}-{role}` | 시맨틱 레이어 |
| Tailwind 유틸리티 | `{prefix}-{element}-{role}` | @theme 매핑 후 자동 생성 |

**매핑 예시**:

| CSS 변수 | Tailwind 유틸리티 | 용도 |
|---------|------------------|------|
| `--text-default` | `text-text-default` | 텍스트 색상 |
| `--surface-error` | `bg-surface-error` | 배경 색상 |
| `--border-warning` | `border-border-warning` | 테두리 색상 |

피드백 색상(error/warning/success)도 일반 토큰과 동일한 구조를 따른다. 별도의 `feedback` 네임스페이스를 만들지 않는다.

**이유**: Tailwind의 `text-*`, `bg-*`, `border-*` 유틸리티와 1:1 대응. `text-text-`를 입력하면 error/warning/success를 포함한 모든 텍스트 색상이 한 목록에 나열된다.

### Brand
서비스의 핵심 정체성 컬러. 사용자의 가장 중요한 행동을 유도한다.
- CTA 버튼, 알림 배지
- 부정적인 활동에 Primary를 사용하지 않는다 (→ Error 사용)

### Neutral
서비스의 뼈대. 텍스트, 배경, 구분선 등 대부분의 UI 요소에 사용된다.

### Status
사용자 행동의 결과나 시스템 상태 변화를 전달한다. 상태와 무관한 UI에는 사용하지 않는다.

### Purple (혜택/긍정적 기회)
쿠폰, 포인트, 할인 등 **사용자에게 긍정적 기회** 를 시각적으로 강조할 때 사용한다.

<Note>

Purple의 의미 확장에 대한 깊은 분석은 Purple 컬러 재정의를 참조한다.

</Note>

## 접근성
모든 색상 조합은 **WCAG AA 레벨 이상의 명도 대비** 를 충족해야 한다.

### 배경색별 권장 전경색 조합
| 배경 토큰 | 권장 전경색 | 비고 |
| --- | --- | --- |
| `bg-surface-default` (`--white`), `bg-surface-secondary` (`--gray-50`) | `text-text-default`, `text-text-secondary`, `text-text-error` 등 **어두운 계열** | 가장 범용적인 조합 |
| `--brand-500` 배경 | `text-text-inverse` (`--white`) | 브랜드 CTA 버튼 |
| `bg-surface-inverse` (`--gray-900`) | `text-text-inverse` (`--white`), `text-text-disabled-subtle` (`--gray-200`) | 다크 섹션, 툴팁 |
| `bg-surface-success` (`--success-50`) | `text-text-on-success` (`--success-700`), 아이콘 `text-icon-on-success` (`--success-600`) | 성공 피드백 패널 |
| `bg-surface-warning` (`--warning-50`) | `text-text-on-warning` (`--warning-700`), 아이콘 `text-icon-on-warning` (`--warning-600`) | 경고 피드백 패널 |
| `bg-surface-error` (`--error-50`) | `text-text-on-error` (`--error-700`), 아이콘 `text-icon-on-error` (`--error-600`) | 오류 피드백 패널 |
| `bg-surface-info` (`--blue-50`) | `text-text-on-info` (`--blue-700`), 아이콘 `text-icon-on-info` (`--blue-600`) | 정보 피드백 패널 |

## Do / Don't

<Check>**Do**</Check>

- 밝은 배경(`--white`, `--gray-50`)에는 `--gray-700` 이상의 어두운 텍스트를 사용한다.
- 피드백 배경(`surface-{status}`, -50)에는 반드시 `text-on-{status}` 토큰(-700)을 사용한다. 흰 바탕에서는 기본 `text-{status}` 토큰(-600)을 사용한다.
- 브랜드 컬러 배경(`--brand-500`)에는 `text-text-inverse` 텍스트만 사용한다.

<Warning>**Don't**</Warning>

- 밝은 배경에 `--gray-500` 이하의 텍스트를 주요 콘텐츠로 사용하지 않는다 (대비 비율 미달).
- 피드백 배경에 gray 계열 텍스트를 사용하지 않는다 (상태의 의미가 희석됨).
- 브랜드 컬러 배경에 gray 계열 텍스트를 사용하지 않는다 (가독성 저하).
- 서로 다른 상태 색상을 배경-전경으로 혼합하지 않는다 (예: error 배경 + success 텍스트).
