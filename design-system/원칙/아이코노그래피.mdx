## 철학
SAZO의 아이코노그래피는 사용자가 인터페이스를 빠르고 직관적으로 이해할 수 있도록 돕는 시각적 언어다. 장식적 요소가 아닌, 명확한 의미를 전달하는 기능적 도구로서 **Lucide 아이콘 라이브러리** 를 기반으로 서비스 전반에 통일된 시각적 경험을 제공한다.

- **명확성 (Clarity)** : 아이콘은 즉시 알아볼 수 있어야 한다. 복잡하거나 추상적인 표현을 피하고, 보편적으로 이해되는 상징을 사용한다.
- **일관성 (Consistency)** : 모든 아이콘은 동일한 시각적 규칙(선 두께, 모서리 등)을 공유하여 통일된 인상을 준다. 하나의 아이콘은 하나의 의미 그룹 내에서만 사용한다. 같은 아이콘이 맥락에 따라 다른 의미로 쓰이면 사용자의 학습이 깨지고, 같은 의미에 다른 아이콘이 쓰이면 인지 부하가 높아진다.
- **간결성 (Simplicity)** : 의미 전달에 불필요한 시각적 요소를 제거한다. 가장 본질적인 형태만으로 아이콘을 표현한다.

<Note>

구체적인 아이콘-의미 매핑 사전은 아이콘 어휘를 참조한다. 이 문서는 원칙과 패턴에, 어휘 사전은 개별 아이콘 룩업에 집중한다.

</Note>

## 시각적 스타일
모든 아이콘은 Lucide의 스타일 가이드를 따른다. 기존 Lucide 아이콘을 사용하거나, 자체 제작할 때 모두 동일하게 적용되는 규칙이다.

| 속성     | 규칙         | 설명                                             |
| ------ | ---------- | ---------------------------------------------- |
| 그리드 크기 | 24x24px    | 모든 아이콘은 24x24px 그리드 위에서 디자인되어 일관된 크기와 비율을 유지한다 |
| 선 두께   | 2px        | 너무 얇지도, 두껍지도 않은 명확한 가시성을 제공한다                  |
| 모서리    | 2px Radius | 부드러운 라운딩 값을 적용하여 친근한 인상을 준다                    |
| 끝점     | Rounded    | 모든 선의 끝점은 둥글게 처리하여 스타일에 통일성을 부여한다              |
| 형태     | 기하학적, 단순함  | 원, 사각형, 직선 등 기본적인 기하학적 도형을 기반으로 구성한다           |
| 관점     | 정면, 2D     | 왜곡이나 원근감이 없는 정면 뷰를 기본으로 한다                     |

## 아이콘 크기 스케일
아이콘은 4가지 크기 스케일로 사용된다.

| 크기 | 용도 | 비고 |
|---|---|---|
| 20px | 소형 | 컴팩트한 목록 아이콘, FeaturedIcon 기본 크기 |
| 24px | 기본 | 독립 아이콘 버튼, 목록 아이템 대표 아이콘, 기본 사용 크기 |
| 40px | FeaturedIcon sm | FeaturedIcon 소형 변형 |
| 48px | FeaturedIcon md | FeaturedIcon 중형 변형 |

<Note>

12px 아이콘은 공간이 매우 협소하고 보편적으로 인지 가능한 단순한 형태에 한해 제한적으로 사용한다. 14px, 16px도 텍스트 보조 역할로 사용되나, 이는 아래 크기-텍스트 조합 가이드에서 다룬다.

</Note>

## 크기-텍스트 조합 가이드
아이콘의 크기는 주변 텍스트와의 관계에 따라 결정된다. 일관된 시각적 리듬을 위해 다음 원칙과 조합을 따른다.

### 핵심 원칙
1. **독립적 사용** : 아이콘 자체로 충분한 의미를 전달할 때는 24px를 기본으로 사용한다. (예: 아이콘 버튼)
2. **동등 계층** : 텍스트와 아이콘이 동등한 중요도를 가질 때, 아이콘의 크기는 텍스트의 행간과 동일하게 설정하여 시각적 무게감을 맞춘다.
3. **보조 계층** : 아이콘이 텍스트를 보조하는 역할을 할 때, 아이콘의 크기는 텍스트의 크기와 동일하게 설정하여 텍스트의 흐름을 방해하지 않는다.
4. **제한적 사용** : 12px 아이콘은 공간이 매우 협소하고, 보편적으로 인지 가능한 단순한 형태에 한해 제한적으로 사용한다.

### 조합 가이드
타이포그래피 정의에 따른 텍스트와 아이콘의 권장 조합이다.

| 텍스트 (토큰/크기) | 계층 | 아이콘 크기 | 주 사용처 |
|---|---|---|---|
| — | 독립적 | 24px | 아이콘 버튼, 텍스트 없는 네비게이션, FeaturedIcon |
| `text-body-default` / `text-label-large` (16px) | 동등 | 24px | 목록 아이템의 대표 아이콘, 주요 기능 설명 |
| `text-body-default` / `text-label-large` (16px) | 보조 | 16px | 본문 내 텍스트를 보조하는 정보 아이콘 |
| `text-body-small` / `text-label-default` (14px) | 동등 | 20px | 컴팩트한 목록 아이템의 대표 아이콘, FeaturedIcon 기본 크기 |
| `text-body-small` / `text-label-default` (14px) | 보조 | 14px | 입력 필드 내부 아이콘, 보조 텍스트 옆 정보 아이콘 |
| `text-body-xs` / `text-label-small` (12px) | 보조 | 12px | 태그(Tag) 내부의 작은 아이콘, 제한적 사용 |

## 색상
아이콘의 색상은 [색상](/design-system/원칙/색상) 문서의 원칙을 따른다.

텍스트와 아이콘이 함께 사용될 때, 아이콘은 동일한 색상의 텍스트보다 시각적으로 더 무겁게 느껴진다. 이를 보정하기 위해 **바로 아래 짝수백 Gray 색상** 을 아이콘에 적용하여 시각적 균형을 맞춘다.

| 텍스트 | 동반 아이콘 | 예시 |
|---|---|---|
| `text-text-default` (gray-900) | `text-text-default-subtle` (gray-800) | 주문 목록 제목 + 아이콘 |
| `text-text-secondary` (gray-700) | `text-text-secondary-subtle` (gray-600) | 모달 설명 + 보조 아이콘 |

## 의미 패턴

<AccordionGroup>

<Accordion title="Chevron vs Arrow">

같은 "오른쪽" 방향이라도 시각적 무게에 따라 역할이 다르다.

| 유형 | 형태 | 역할 | 사용처 |
|---|---|---|---|
| **Chevron** | 가벼움 (›) | 인라인 힌트 — "여기 갈 수 있어요" | 메뉴 항목 진입, 더보기 링크, 아코디언, 캐러셀 |
| **Arrow** | 무거움 (→) | 액션 버튼 — "여기 눌러서 가세요" | CTA 버튼, 페이지네이션, submit, 돌아가기 |

**판단 기준:**
- 인터랙티브 요소에 **장식적으로 붙는** 방향 표시 → Chevron
- 버튼 안에서 **행동을 지시하는** 방향 표시 → Arrow
- 같은 화면에서 동일 의미에 Chevron과 Arrow를 혼용하지 않는다

</Accordion>

<Accordion title="상태 피드백: 형태별 시맨틱">

아이콘의 외곽 형태가 상태의 심각도를 나타낸다.

| 형태      | 아이콘             | 의미                | 근거                                                  |
| ------- | --------------- | ----------------- | --------------------------------------------------- |
| **원형**  | LuCircleAlert   | error — 이미 문제가 발생 | 닫힌 형태 = 확정적 상태. 업계 관행 (Material, Carbon, Atlassian) |
| **삼각형** | LuTriangleAlert | warning — 주의가 필요  | ISO 7010 경고 기호. 뾰족함 = 위험 암시, 아직 발생하지 않은 것           |

**조합 규칙:**
- 형태 + 색상 + 텍스트 3요소를 항상 함께 사용한다 (색상만으로 상태를 구분하지 않는다)
- error: LuCircleAlert + `text-text-error` 색상
- warning: LuTriangleAlert + `text-text-warning` 색상
- 하나의 아이콘을 여러 시맨틱 레벨에 걸쳐 사용하지 않는다

</Accordion>

<Accordion title="주문 상태: Clipboard 패밀리">

주문의 생명주기를 Clipboard 계열 아이콘으로 표현한다. 기본 형태(클립보드)는 "주문서"를 상징하고, 수식 요소가 상태를 구분한다.

| 수식 요소 | 아이콘 | 의미 | 설명 |
|---|---|---|---|
| 펜 (Pen) | LuClipboardPen | 주문 대기 | 주문서를 작성 중 |
| 체크 (Check) | LuClipboardCheck | 주문 완료 | 주문서가 확정됨 |
| 리스트 (List) | LuClipboardList | 주문 변경 | 주문서 내용이 바뀜 |
| 마이너스 (Minus) | LuClipboardMinus | 단순 취소 | 사용자가 주문을 줄임 |
| X | LuClipboardX | 주문 불가/거절 | 시스템이 주문을 막음 |

**Minus vs X 판단 기준:**
- **사용자가 자발적으로 줄인 것** → Minus (부분 취소, 수량 감소)
- **시스템/관리자가 막은 것** → X (품절 거절, 정책 불가)

</Accordion>

<Accordion title="배송 단계: 이동 수단 아이콘">

구매대행 물류 흐름을 이동 수단의 규모로 단계를 구분한다.

```
주문 확인 → 판매처 주문 → 입고/검품 → 통관 → 국제 배송 → 국내 배송 → 배송 완료
  Check     ClipboardCheck  PackageOpen  SearchCheck  Plane      Car       PackageCheck
```

| 원칙 | 설명 |
|---|---|
| 이동 수단 스케일 | 비행기(국제) > 자동차(국내) — 운송 범위와 아이콘 스케일이 대응 |
| 상자 상태 | 열림(PackageOpen) = 확인 중, 닫힘+체크(PackageCheck) = 수령 완료 |
| 단계 전용 | 각 물류 단계에 지정된 아이콘은 해당 단계에서만 사용. LuTruck을 특정 물류 단계에 쓰지 않는다 |

</Accordion>

<Accordion title="추상화 수준: 같은 아이콘의 다중 사용 허용 기준">

하나의 아이콘이 여러 곳에서 쓰일 때, "같은 의미인가 다른 의미인가"를 판단하는 기준이다.

| 수준 | 허용 여부 | 예시 |
|---|---|---|
| 같은 추상 의미 | 허용 | LuChevronDown이 "아코디언 접힘"과 "순위 하락" 양쪽에 사용 — 둘 다 "아래 방향"이라는 추상 의미 |
| 같은 추상 의미 | 허용 | LuMegaphone이 "공지", "회원가입 알림", "약관변경 알림"에 사용 — 모두 "시스템이 알린다"는 추상 의미 |
| 다른 추상 의미 | 금지 | LuPackage가 "배송 완료"와 "주문제작"에 사용 — "닫힌 상자"와 "제작"은 다른 추상 의미 |
| 다른 추상 의미 | 금지 | LuX가 "닫기/삭제"와 "1개 제한"에 사용 — "제거"와 "제한"은 다른 추상 의미 |

**판단 질문:** "이 두 사용처를 한 문장으로 묶을 수 있는가?"
- "아코디언 접힘과 순위 하락은 둘 다 **아래 방향** 이다" → 묶임 (허용)
- "배송 완료와 주문제작은 둘 다 **???** 이다" → 안 묶임 (금지)

</Accordion>

</AccordionGroup>

## 아이콘 선택 절차

<Steps>

<Step title="아이콘 어휘 사전 확인">

해당 의미에 이미 승인된 아이콘이 있는지 먼저 확인한다. 의미별 승인 아이콘 목록은 AI 컨텍스트 문서에서도 참조할 수 있다.

</Step>

<Step title="Lucide 라이브러리 탐색">

어휘 사전에 없는 새로운 의미라면, [Lucide 아이콘 라이브러리](https://lucide.dev/icons/)에서 적합한 아이콘을 찾는다.

</Step>

<Step title="어휘 사전에 등록">

선택한 아이콘과 의미를 어휘 사전에 추가한 후 코드에 적용한다.

</Step>

<Step title="자체 제작 요청">

Lucide에도 적합한 아이콘이 없으면, 아래 절차에 따라 제작을 요청한다.

</Step>

</Steps>

## 자체 제작 요청 절차

<Steps>

<Step title="필요성 정의">

어떤 맥락에서, 어떤 의미를 전달하기 위해 아이콘이 필요한지 명확하게 정의한다.

</Step>

<Step title="레퍼런스 수집">

전달하려는 의미와 유사한 다른 아이콘 레퍼런스를 2~3개 수집한다.

</Step>

<Step title="요청">

위 내용을 정리하여 디자인팀에 제작을 요청한다. 제작된 아이콘은 위의 시각적 스타일 규칙(24px 그리드, 2px stroke, 2px radius, rounded terminals)을 준수해야 한다.

</Step>

</Steps>

## 접근성
- 의미를 전달하는 모든 아이콘은 스크린 리더 사용자를 위해 명확한 텍스트 라벨(`aria-label` 등)을 반드시 제공해야 한다.
- 장식적인 목적의 아이콘은 스크린 리더가 읽지 않도록 처리한다 (`aria-hidden="true"`).
