## 철학
SAZO의 엘리베이션 시스템은 **시각적 깊이(depth)** 를 통해 UI 요소 간의 관계와 중요도를 전달한다.

- **물리적 은유** : 그림자의 크기와 흐림이 "높이"를 나타낸다. 높을수록 사용자 주의를 더 끈다.
- **점진적 스케일** : none → lowest → low → medium → high → higher → highest → overlay. 단계를 건너뛰지 않는다.
- **기능적 목적** : 장식이 아닌 계층 전달. 모든 그림자는 "이 요소가 왜 떠 있는가"에 대한 답이 있어야 한다.
- **일관된 색조** : 기본 그림자는 모두 `rgba(16, 24, 40, *)` 기반으로 중립적 깊이감을 유지한다.

토큰 정의: `02 토큰/css/tokens.css`, 복합 유틸리티: `02 토큰/css/utilities.css`

## Elevation 복합 유틸리티
`utilities.css`에서 shadow + z-index를 조합한 복합 클래스를 제공한다.

| 유틸리티 클래스 | Shadow | Z-index | 용도 |
|---------------|--------|---------|------|
| `elevation-modal` | `--shadow-overlay` | `--z-modal` | SazoModal, SectionBottomModal |
| `elevation-dropdown` | `--shadow-highest` | `--z-dropdown` | 드롭다운 패널 |
| `elevation-dock` | `--shadow-dock` | `--z-dock` | 모바일 하단 독 |
| `elevation-header` | `--shadow-medium` | `--z-header` | PC/모바일 헤더 |

## Shadow 구조 패턴
기본 계층(`--shadow-lowest` ~ `--shadow-highest`)은 일관된 설계 패턴을 따른다:
- **색상** : `rgba(16, 24, 40, *)` -- gray-900 기반의 중립색
- **레이어** : 대부분 2-layer. 첫 번째는 가까운 그림자(낮은 blur), 두 번째는 먼 그림자(높은 blur)
- **opacity** : 단계가 올라갈수록 두 번째 레이어의 opacity가 증가 (0.05 → 0.10 → 0.10 → 0.08 → 0.08 → 0.14)
- **음수 spread** : `--shadow-medium`부터 음수 spread를 사용하여 그림자 범위를 제어

## Z-index 계층
| CSS 변수 | 값 | 용도 |
|---------|-----|------|
| `--z-timeline` | 10 | 타임라인 아이템 |
| `--z-loading-overlay` | 20 | 주문 로딩 오버레이 |
| `--z-dock` | 40 | 모바일 하단 독 |
| `--z-header` | 100 | PC/모바일 헤더, 서브 네비게이션 |
| `--z-modal` | 999 | SazoModal, SectionBottomModal |
| `--z-header-inner` | 1,000 | 헤더 내부 요소 (GNB 바) |
| `--z-overscreen` | 9,999 | 전체 화면 로딩 |
| `--z-search-history` | 10,000 | 검색 히스토리 패널 |
| `--z-dropdown` | 100,000 | 드롭다운 패널 |
| `--z-external-overlay` | 99,999,999 | 외부 본인인증 모달(다날) |

### Z-index 설계 원칙
- **10배 간격** : 각 계층 사이에 충분한 여유 공간을 둔다. 새로운 계층이 필요해도 기존 값을 변경하지 않는다.
- **Shadow와 Z-index의 대응** : 높은 elevation은 높은 z-index를 동반하는 경향이 있지만, 반드시 1:1 대응하지는 않는다. `shadow-medium`이 `--z-header`(100)를 사용하고, `shadow-highest`가 `--z-dropdown`(100,000)을 사용한다.
- **외부 서비스 대응** : `--z-external-overlay`(99,999,999)는 다날 본인인증 등 외부 서비스가 요구하는 최상위 계층이다. 자체 UI에서는 사용하지 않는다.
