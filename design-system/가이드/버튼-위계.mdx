## 철학
### 왜 브랜드 컬러를 아끼는가
브랜드 컬러(`brand-500`)는 사용자의 무의식 속에 서비스의 감정적 이미지를 형성한다. "삭제하시겠습니까?" 같은 부정적 맥락에 반복 노출되면 브랜드 자체에 부정적 연상을 학습하므로, 긍정적 기회에만 브랜드 컬러를 사용하고 일반 확인에는 Gray-800, 파괴적 액션에는 Error 컬러를 사용한다. Purple 컬러 재정의에서 다룬 색상 의미론의 연장선이며, Airbnb의 접근을 참조하되 역직구 커머스 맥락에 맞게 조정했다.
primary에서 brand로의 전환 참조.

### 계층과 색상의 분리
버튼의 색상은 "얼마나 중요한가"가 아니라 "어떤 성격의 액션인가"가 결정한다. 계층(hierarchy)과 색상(color)을 분리하여, Primary 계층의 버튼이 맥락에 따라 브랜드 컬러일 수도, 그레이일 수도, 에러 컬러일 수도 있다.

## 의사결정 트리
색상 선택이 필요할 때 아래 트리를 순서대로 따른다.

```
이 액션은 무엇인가?
├── 파괴적인가? (삭제, 탈퇴, 취소불가)
│   └── → Destructive (error-600)
├── 강한 긍정인가? (구매, 카트담기, 쿠폰적용, 결제)
│   └── → Brand (brand-500)
├── 혜택/프로모션인가?
│   └── → Benefit (purple) — 매우 제한적
├── 일반 확인/진행인가?
│   ├── 화면에 gray-800 버튼이 이미 많은가?
│   │   ├── Yes → Light (gray-300 계열) 로 무게 감소
│   │   └── No → Default (gray-800)
│   └── → Default (gray-800)
└── 보조/취소인가?
    ├── "취소", "돌아가기" → Cancel (Link)
    └── 그 외 보조 → Outline (white+border)
```

## 버튼 구성
### 6가지 색상 변형
6가지 색상 변형은 각각 고유한 맥락과 사용 빈도를 갖는다.

| 색상 변형 | 배경 | 텍스트 | 맥락 | 빈도 |
|-----------|------|--------|------|------|
| **Brand** | brand-500 (primitive 직접) | `--text-inverse` | 강한 긍정 CTA | 제한적 |
| **Default** | gray-800 (primitive 직접) | `--text-inverse` | 일반 확인/진행 | **가장 빈번** |
| **Light** | gray-300 계열 (primitive 직접) | `--text-default` | Default가 과밀할 때 무게 감소 | 상황적 |
| **Outline** | `--surface-default` | `--text-default` | 보조 액션, 취소 | 빈번 |
| **Destructive** | error-600 (primitive 직접) | `--text-inverse` | 삭제, 탈퇴 | 제한적 |
| **Benefit** | purple-500 (primitive 직접) | `--text-inverse` | 혜택/프로모션 전용 | 매우 제한적 |

<Note>
  버튼 배경색은 시맨틱 토큰이 아닌 **컴포넌트 레벨에서 primitive를 직접 참조** 한다. 이는 action 시맨틱 카테고리를 제거한 설계 결정에 따른 것이다.
</Note>

**+ Cancel (Link 스타일)** : 버튼이 아닌 텍스트 링크. "취소", "돌아가기" 전용.

### 일상적으로 사용하는 4가지
대부분의 UI는 아래 4가지만으로 구성된다:

```
[Default (gray-800)]     — 일반 확인, 저장, 다음
[Outline (white+border)] — 보조 액션, 이전
[Brand (brand-500)]      — 카트 추가, 구매, 쿠폰 적용
취소 (Link)              — 돌아가기
```

Destructive와 Benefit은 특수 맥락에서만 등장한다.

## Do / Don't
| Do | Don't |
|----|-------|
| 브랜드 컬러는 긍정적 CTA에만 | 일반 확인에 브랜드 컬러 사용 |
| 한 화면에 Brand 버튼은 1개 | 여러 Brand 버튼으로 주의 분산 |
| gray-800이 3개 이상 반복되면 Light로 전환 | gray-800 5개를 나란히 배치 |
| 파괴적 액션에 Destructive 색상 | "삭제"에 Brand 컬러 (부정적 학습) |
| 취소/돌아가기는 항상 Link 스타일 | 취소에 Filled 버튼 (시각적 경쟁) |
| Outline은 Secondary 계층에만 | Primary 액션에 Outline |
| Benefit(purple)은 프로모션에만 극소량 | 일반 UI에 purple 버튼 |

## 설계 판단 기록
<AccordionGroup>
  <Accordion title="Brand 호버가 그라데이션인 이유">
    단순 `brand-600` 호버는 색상 변화가 미세하여 피드백이 약하다. Airbnb 스타일의 그라데이션 호버는 더 풍부한 시각적 피드백을 제공하면서도 브랜드 톤을 유지한다. 이 로직은 시맨틱 토큰이 아닌 컴포넌트 레벨에서 처리한다.

  </Accordion>
  <Accordion title="gray-800을 \"기본 Primary\"로 선택한 이유">
    흰 배경 위에서 충분한 대비와 시각적 무게를 제공하면서도, 브랜드 컬러의 감정적 의미를 소진하지 않는다. "중립적이면서 강한" 색상이 필요한 대부분의 버튼에 적합하다.

  </Accordion>
  <Accordion title="Light 변형의 사용 기준">
    정량적 기준: 한 뷰포트(스크롤 없이 보이는 영역) 안에 gray-800 버튼이 **3개 이상** 반복되면 시각적 무게가 과도해진다. 이때 중요도가 낮은 버튼부터 Light로 전환한다.

  </Accordion>
</AccordionGroup>
