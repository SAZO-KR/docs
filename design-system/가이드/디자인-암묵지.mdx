디자이너의 암묵적 설계 규칙을 명시화한 문서. 디자이너·개발자·AI 모두가 참조하여 일관된 설계 판단을 내릴 수 있도록 한다. 이 문서의 내용은 엄격한 "규칙"이 아니라 "이런 맥락에서 이런 판단을 하는 경향"이다. 예외가 필요하면 맥락에 맞게 판단하되, 이 문서의 논리를 이해한 위에서 벗어날 것.

<Note>

이 문서는 피그마 작업물과 코드베이스에서 역추출한 패턴을 바탕으로 작성됨. 암묵지가 추가되면 계속 갱신한다.

</Note>

## 범용 원칙

<AccordionGroup>

<Accordion title="근접성 법칙 기반 간격 체계">

하위 그룹으로 들어갈수록 이전 그룹 간격보다 같거나 작아야 한다.

```
섹션 간 여백: 32px
  → 섹션 내 그룹 간: 24px
    → 그룹 내 요소 간: 16px
      → 요소 내 컬럼 간: 12px
```

이 비율은 절대값이 아니라 **상대적 관계** . 섹션 간이 48px면 그룹 간은 32px, 요소 간은 24px 식으로 비례 유지.

</Accordion>

<Accordion title="타이포그래피 기본값">

- 특별한 이유가 없으면 **16px부터 시작** . 중요한 건 16px보다 크게.
- 페이지 밀도가 높으면 14px를 기본으로 잡을 수 있음 — 단, "본문"이 아니라 **UI 라벨/디스크립션에 한해서** .
- 본문 텍스트는 항상 16px 유지.

</Accordion>

<Accordion title="경계색 패턴">

- 카드 등의 경계색은 **gray-300** 기본.
- 엘리베이션(shadow)이 적용된 카드는 경계색을 **gray-200** 으로 낮춤 — shadow가 이미 시각적 구분을 하므로 border는 미세하게.

</Accordion>

<Accordion title="UI Delimiter">

#### / (슬래시)
한 행 내에서 텍스트 요소 간 분리. 행을 나누기보다 한 행에서 처리하는 게 좋지만 dot보다 구분이 강할 때.

</Accordion>

<Accordion title="컴포넌트별 규칙">

각 컴포넌트 문서에 흡수됨. 상세 내용은 해당 문서 참조.

- **버튼** — 크기 선택(lg/md/sm = 페이지/섹션/하위섹션), 정렬, 바텀 버튼 컨테이너 → [button](/design-system/컴포넌트/button)
- **타일 버튼** — 사용 시점(위계 소진 + 텍스트 길이), Disabled 없음, Chevron 어포던스 → [tile-button](/design-system/컴포넌트/tile-button)
- **타일 셀렉터** — 사용 조건, vs 타일 버튼 구분, vs 리스트 경계 → [tile-selector](/design-system/컴포넌트/tile-selector)
	- 설계 기록: 타일 셀렉터와 타일 버튼의 차이
- **인포패널** — 크기 선택(md-full=페이지, xs=뱃지 확장) → [info-panel](/design-system/컴포넌트/info-panel)
- **디바이더** — 여백 규칙, Thin/Thick 사용 기준 → [divider](/design-system/컴포넌트/divider)
- **피쳐드 헤딩** — 48px 표준, 중앙 정렬, EmptyState 패턴 → [featured-icon](/design-system/컴포넌트/featured-icon)

</Accordion>

<Accordion title="페이지/레이아웃 규칙">

각 레시피/가이드 문서에 흡수됨. 상세 내용은 해당 문서 참조.

- **페이지 배경색** — 결제 맥락에서만 PC gray 배경 → [섹션 조합](/design-system/레시피/섹션-조합)
- **페이지 최대 너비** — 400/480/640/720/734 → [섹션 조합](/design-system/레시피/섹션-조합)
- **GNB** — 모바일 뒤로가기 + 2 addon → [섹션 조합](/design-system/레시피/섹션-조합)
- **이벤트 페이지** — 128px 섹션 간격 → [섹션 조합](/design-system/레시피/섹션-조합)
- **카드 디자인** — 패딩-라운드 매칭, PC→MO 전환 → [카드 조합](/design-system/레시피/카드-조합)
- **모달** — 바텀시트 기본, 아나토미, 타이틀 생략 3조건 → [모달 조합](/design-system/레시피/모달-조합)
- **더보기 버튼** — 섹션 하단 중복 배치 → [인터랙션 패턴](/design-system/가이드/인터랙션-패턴)

</Accordion>

</AccordionGroup>
