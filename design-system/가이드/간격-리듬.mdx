## 철학
SAZO의 간격 체계는 **4px 기반 스케일** 과 세 가지 리듬 원칙으로 구성된다.

1. **섹션 간격 48px (`mb-12`)** : 페이지 수직 리듬의 기본 박자
2. **제목-콘텐츠 간격 16px (`mb-4`)** : SectionTitle 아래의 표준 간격
3. **밀접한 요소 8px (`gap-2`)** : 같은 의미 단위의 내부 간격

## 간격 스케일
| 값 | Primitive | 사용처 |
|------|-----------|--------|
| 4px | `gap-1` | 극소 인라인, 타이틀-서브타이틀 |
| 8px | `gap-2` | 인라인 옵션, 텍스트 보조, 리스트 카드 간 |
| 12px | `gap-3` | 모바일 카드 열 간격, 소형 패딩 |
| 16px | `gap-4` | 기본 패딩, 텍스트 블록 간, 제목-콘텐츠 |
| 20px | `mt-5` | FeaturedHeading 텍스트 간, 모바일 화면 가장자리(고정) |
| 24px | `gap-6` | PC 카드 열 간격, 소형 섹션 간 |
| 28px | `gap-7` | 폼 필드 간 간격 |
| 32px | `p-8` | 대형 컴포넌트 패딩, 중간 섹션 간격 |
| 40px | `mb-10` | FeaturedHeading 하단, 환불 섹션 간격 |
| 48px | `mb-12` | 메인 섹션 간격, PC 화면 가장자리 |
| 64px | `py-16` | 대형 섹션 패딩, 모달 오버레이 |

## Do / Don't

<Check>

**Do**

메인 섹션 간격 `mb-12` (48px) 통일

SectionTitle 아래 `mb-4` (16px)

FeaturedHeading 하단 `mb-10` (40px)

독립 폼 필드 간격 `gap-7` (28px)

밀접한 요소 내부 `gap-2` (8px)

콘텐츠 영역은 `<Spacer>` 컴포넌트

Mobile 오프셋은 CSS 변수 기반 유틸리티

`padding`은 Wrapper(단일 자식) 책임, `gap`은 Container(다중 자식) 책임 — 분리

뷰포트 gutter는 `<Spacer>`·Page Shell에 위임

</Check>

<Warning>

**Don't**

섹션마다 다른 간격 (mb-8, mb-16 혼용)

SectionTitle 아래 `mb-2`나 `mb-8`

FeaturedHeading 하단 `mb-12`

폼 필드에 `gap-4`나 `gap-8`

내부 요소에 `gap-4` 이상

직접 `px-*` 클래스 개별 적용

`mt-[100px]` 하드코딩

같은 요소에 `p-*`와 `gap-*` 동시 사용

본문 내부 래퍼에 `md:px-0` 같은 "패딩 지우는" 조건부 유틸

</Warning>

<Note>

Wrapper / Container 어휘 구분과 판단 트리는 레이아웃 프리미티브 어휘 참조. 결정 배경은 레이아웃 프리미티브 설계 판단.

</Note>
