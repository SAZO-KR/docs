### WCAG 매핑표
UX Writing 관점에서 적용되는 WCAG 기준과 SAZO 대응 방안을 매핑한다.

### Guideline 3.1: 읽을 수 있음
| 기준 | 적합 수준 | SAZO 대응 |
|------|-----------|-----------|
| 3.1.3 일반 단어 사용 | AAA | 전문 용어 사용 시 괄호 또는 툴팁으로 설명 제공. 첫 사용 시 약어를 풀어쓴다 |
| 3.1.4 약어 | AAA | 약어 첫 사용 시 풀어서 표기 (예: QR(Quick Response) 코드) |
| 3.1.5 읽기 수준 | AAA | 중학교 수준 읽기 능력으로 이해 가능하도록 작성 |

### Guideline 3.2: 예측 가능성
| 기준 | 적합 수준 | SAZO 대응 |
|------|-----------|-----------|
| 3.2.1 초점 | A | 버튼 텍스트가 클릭 결과를 예측할 수 있어야 한다 |
| 3.2.2 입력 시 | A | 라벨과 도움말이 행동 결과를 명확히 설명한다 |
| 3.2.3 일관된 네비게이션 | AA | 메뉴·버튼 레이블을 서비스 전체에서 동일하게 유지한다 |
| 3.2.4 일관된 식별 | AA | 같은 기능에는 같은 텍스트를 사용한다 |

### Guideline 3.3: 입력 보조
| 기준 | 적합 수준 | SAZO 대응 |
|------|-----------|-----------|
| 3.3.1 에러 식별 | A | 에러 위치와 원인을 명확히 전달하는 메시지 제공 |
| 3.3.2 라벨 또는 설명 | A | 모든 입력 필드에 시각적 라벨과 프로그래밍 라벨 제공 |
| 3.3.3 에러 제안 | AA | 에러 발생 시 구체적 수정 방안 제시 |
| 3.3.4 에러 방지 | AA | 결제 등 중요 작업은 확인 단계를 포함 |
| 3.3.5 도움말 | AAA | 입력 형식·제약 조건을 도움말로 제공 |

### JIS X 8341-3 보충
일본 시장 서비스인 SAZO에 적용되는 일본 접근성 법규 요구사항을 정리한다.

### 법적 배경
- **JIS X 8341-3:2016** : WCAG 2.0 Level AA와 실질적으로 동일한 일본 산업규격
- **장애자차별해소법(障害者差別解消法)** 2024년 4월 개정: 민간 사업자에게도 합리적 배려(合理的配慮) 제공이 법적 의무로 전환
- **금융청 가이드라인** : 금융 서비스 사업자의 웹 접근성 확보를 권고

### SAZO 적합 목표
- Level AA 전체 적합을 기본 목표로 설정
- 결제 서비스 크리티컬 항목은 AAA 기준을 선별 적용 (3.1.3, 3.1.4, 3.1.5, 3.3.5)

### 종합 체크리스트
배포 전 점검과 QA에 사용하는 카테고리별 체크리스트다.

### 명확성
- [ ] 문장이 15~20단어 이하인가?
- [ ] 이중 부정이 없는가?
- [ ] 능동태를 사용했는가?
- [ ] 한 문장이 한 개의 아이디어만 담는가?
- [ ] 전문 용어가 없거나 정의되었는가?
- [ ] 약어가 첫 사용 시 풀어 썼는가?

### 구조와 형식
- [ ] 페이지에 h1이 정확히 하나 있는가?
- [ ] 헤딩이 계층적으로 올바른가? (h1 → h2 → h3)
- [ ] 긴 텍스트가 여러 단락으로 나뉘었는가?
- [ ] 관련 항목이 목록으로 그룹화되었는가?
- [ ] 테이블에 caption과 헤더(scope)가 있는가?

### 접근성 기술
- [ ] 모든 이미지에 alt 텍스트가 있는가?
- [ ] alt 텍스트에 "이미지", "버튼" 등 역할 중복이 없는가?
- [ ] 장식 이미지는 `alt=""` 또는 `aria-hidden="true"`인가?
- [ ] 모든 링크 텍스트가 목적지를 명확히 설명하는가?
- [ ] 폼 입력 필드에 라벨과 `aria-describedby`가 있는가?
- [ ] 에러 표시가 색상+아이콘+텍스트 조합인가?
- [ ] 색상만으로 상태(성공/경고/정보)를 전달하지 않았는가?

### 포용성
- [ ] 젠더 중립 표현을 사용했는가?
- [ ] 장애 관련 용어가 존중하는 표현(사람 우선)인가?
- [ ] 나이, 인종, 문화적 편견이 없는가?
- [ ] 모든 대상 사용자가 환영받는 느낌을 주는가?

### 일관성
- [ ] 같은 개념이 같은 텍스트로 표현되었는가?
- [ ] 버튼 레이블이 서비스 전체에서 일관되는가?
- [ ] 에러 메시지 형식이 일관되는가?

### 결제 서비스 특화 체크리스트
SAZO 결제 플로우에 특화된 접근성 점검 항목이다.

### 금액 표시
- [ ] 스크린리더가 금액을 올바르게 읽는가? (`aria-label`로 "千二百八十円" 등 명시 권장)
- [ ] 할인가/원가/최종가가 텍스트로 구분되는가? (취소선만으로 구분 금지)
- [ ] 통화 단위(JPY/KRW)가 명확히 표시되는가?

### 거래 확인
- [ ] 결제 확정 전 금액/수단/상품을 확인하는 단계가 있는가?
- [ ] 결제 버튼이 명시적 텍스트인가? ("다음" 금지, "¥12,800 결제하기" 권장)
- [ ] 결제 수단/금액 변경 시 변경 내용이 텍스트로 명시되는가?

### 오류 복구
- [ ] 결제 실패 시 실패 원인 + 해결 방법 + 데이터 보존 여부를 모두 안내하는가?
- [ ] 세션 타임아웃 전 사전 경고가 있는가?
- [ ] 부분 실패(주문 접수 but 결제 보류) 상태를 명확히 전달하는가?

### 배송 상태
- [ ] 배송 단계가 프로그레스바만이 아닌 텍스트 레이블로도 표시되는가?
- [ ] 현재 상태에 `aria-current`가 적용되었는가?
