이 문서는 SAZO.shop의 디지털 제품을 만드는 모든 사람을 위한 가이드입니다.

SAZO 디자인 시스템은 단순히 재사용 가능한 UI 컴포넌트의 모음집이 아닙니다. 이것은 우리의 디자인 원칙을 담은 살아있는 시스템이자, SAZO만의 일관된 사용자 경험을 만들기 위한 **공동의 언어** 이며, 더 나은 제품을 더 효율적으로 만들기 위한 **전략적인 도구** 입니다.

이 문서는 기술적 세부 사항보다 상위 레벨인, 우리가 **'왜'** 이러한 방식으로 디자인하는지에 대한 철학과 목적을 공유하는 데 집중합니다.

## 이 시스템의 목적

<AccordionGroup>

<Accordion title="일관된 경험을 위한 명문화">

SAZO의 프로덕트 디자인 리더로서, 그동안 암묵적인 기준과 개인적인 경험을 바탕으로 많은 디자인 결정을 내려왔습니다. 하지만 팀이 성장하고 제품이 복잡해짐에 따라, 이러한 암묵적인 규칙들은 혼란의 원인이 될 수 있습니다.

이 시스템은 그동안 흩어져 있던 패턴, 규칙, 원칙들을 명확하게 **'명문화'** 하는 시도입니다. 이를 통해 우리는 개인의 기억이 아닌, 팀 모두가 공유하는 객관적인 기준 위에서 일관된 품질의 디자인을 만들어나갈 것입니다.

</Accordion>

<Accordion title="새로운 팀원을 위한 나침반">

새로운 프로덕트 디자이너가 팀에 합류했을 때, 가장 어려운 점은 SAZO만의 디자인 철학과 암묵적인 규칙들을 파악하는 것입니다. 이 디자인 시스템은 새로운 팀원을 위한 **'나침반'** 역할을 합니다.

신규 입사자는 이 문서를 통해 단순히 "버튼은 이렇게 써"를 배우는 것이 아니라, "SAZO는 왜 이렇게 디자인하는가?"라는 **철학** 을 먼저 흡수하게 됩니다. 이는 온보딩 과정을 가속화하고, 새로운 팀원이 빠르게 팀의 일원으로서 자신감을 갖고 기여할 수 있도록 돕습니다.

</Accordion>

<Accordion title="효율적인 협업을 위한 공동의 언어">

좋은 제품은 디자이너, 기획자, 개발자 등 여러 팀의 유기적인 협력을 통해 만들어집니다. 이 과정에서 다양한 의견이 오가는 것은 당연하지만, 때로는 개인의 취향이나 주관적인 감각에 의존한 논의가 길어지거나, 전체적인 일관성을 해치는 결정을 내릴 위험이 있습니다.

이 디자인 시스템은 디자이너만을 위한 규칙집이 아니라, 모든 팀원이 동일한 목표를 향해 나아갈 수 있도록 돕는 **'객관적인 기준점이자 공동의 언어'** 입니다.

예를 들어, "이 버튼이 좀 더 눈에 띄었으면 좋겠어요"라는 의견이 나왔을 때, 우리는 "우리의 03 버튼 위계 원칙에 따르면, 사용자의 주요 행동을 유도하는 버튼에는 Brand 색상을 사용하기로 약속했습니다. 이 원칙을 바탕으로 가장 효과적인 해결책을 함께 찾아볼까요?"와 같이, 시스템을 근거로 건설적인 대화를 시작할 수 있습니다.

이는 개인의 주관이 아닌 시스템이라는 공동의 자산을 중심으로 논의를 집중시켜, 더 빠르고 효율적인 의사결정을 가능하게 합니다.

이 시스템의 3대 설계 원칙(정직한 안내자, 본질만 남긴 간결함, 체계적 확장)은 01 디자인 원칙에서 상세히 다룹니다.

</Accordion>

</AccordionGroup>

## 토큰 아키텍처

<AccordionGroup>

<Accordion title="Primitive">

팔레트 원본값. 색상(342개 정의, ~35개 활성 사용), 타이포그래피 스케일, 단위값.
직접 사용하지 않고 시맨틱 토큰을 통해서만 참조한다. 단, 버튼 배경처럼 인터랙티브 요소의 배경색은 컴포넌트 레벨에서 primitive를 직접 참조한다 (03 버튼 위계 참조).

</Accordion>

<Accordion title="Semantic">

역할 기반 변수. "이 색상이 무엇인가"가 아니라 "이 색상이 어디에 쓰이는가"를 표현한다.
카테고리: text, surface, border, overlay.

</Accordion>

<Accordion title="CSS / Tailwind">

시맨틱 토큰을 CSS Custom Properties와 Tailwind `@theme inline`으로 변환.
`02 CSS/tokens.css`에서 3개 레이어를 한 파일로 관리한다.

</Accordion>

</AccordionGroup>

## 네이밍 규칙

<AccordionGroup>

<Accordion title="색상 토큰 — Element-first">

```
color.{element}.{role}

color.text.default        ← 요소(text)가 먼저
color.surface.error       ← 피드백도 동일 구조
color.border.warning      ← 별도 feedback 네임스페이스 없음
```

</Accordion>

<Accordion title="타이포그래피 토큰">

```
type.{category}.{variant}

type.title.section.lg     ← 중-대형 섹션 타이틀
type.body.default         ← 기본 본문
type.label.small          ← 소형 라벨 (배지 포함)
```

</Accordion>

<Accordion title="간격 토큰">

```
spacing.{context}.{size}

spacing.screen-edge.pc    ← PC 화면 가장자리 (48px)
spacing.screen-edge.mobile ← Mobile 화면 가장자리 (20px)
spacing.between-sections.xl ← 섹션 간 최대 간격
```

</Accordion>

<Accordion title="Elevation 토큰">

의미 기반: none, lowest, low, medium, high, higher, highest, overlay, dock

</Accordion>
