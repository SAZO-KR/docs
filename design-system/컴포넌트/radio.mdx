### 언제 사용하는가
상호 배타적인 옵션 그룹에서 **반드시 하나만** 선택해야 할 때 사용한다. 옵션 2~5개 범위에서 가장 효과적이다.

### 대안 분기표
| 상황 | Radio 대신 사용할 컴포넌트 | 이유 |
|------|--------------------------|------|
| 다중 선택이 필요한 옵션 | Checkbox | Radio는 하나만 선택 가능 |
| 옵션이 하나뿐인 경우 | Checkbox 또는 Toggle | 단일 옵션은 선택/해제 UI |
| 옵션이 6개 이상 | Dropdown (Select) | Radio가 많으면 화면을 과도하게 차지 |
| 복합 콘텐츠가 포함된 옵션 선택 | TileSelector | 이미지+텍스트+가격 등 복합 레이아웃에는 TileSelector |
| 필터/태그 토글 | Chip | 필터 맥락에서는 Chip 그룹이 공간 효율적 |

### 선택 가이드
### 색상 선택
| Color | 사용 맥락 | 예시 |
|-------|-----------|------|
| `brand` | 주요 선택 플로우, 결정적 선택 | 배송 방법, 결제 수단, 구매 옵션 |
| `black` | 부가 설정, 관리 옵션, 필터 | 정렬 기준, 표시 방식, 부가 설정 |

- `brand`는 brand-500 (primitive 직접) 배경으로 브랜드 색상을 전달한다.
- `black`은 gray-800 (primitive 직접) 배경으로 중립적 톤을 전달한다.
- 같은 폼 내에서 색상을 무작위로 섞지 않는다. 색상이 선택의 중요도를 전달하므로 일관성이 필요하다.

### 크기 선택
| Size | 크기 | 사용 맥락 | 배치 위치 |
|------|------|-----------|-----------|
| `sm` | 16px | 밀집 리스트, 테이블 내부, 사이드바 | 테이블 행, 좁은 설정 패널 |
| `md` | 20px | 기본 라디오, 폼 내부 옵션 | 배송 옵션, 결제 수단, 설정 페이지 |
| `lg` | 24px | 강조 선택, 모바일 터치 최적화 | 모바일 풀 너비 옵션, 온보딩 |

### 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 수직 옵션 목록 (기본) | Radio+라벨 행을 위에서 아래로 나열 | 행 간 gap 12~16px |
| 수평 옵션 배치 | Radio+라벨을 나란히 수평 배치 | 옵션 간 gap 16~24px |
| 카드 내부 옵션 | 카드/패널 내에서 배송 방법 등 선택 | 행 간 gap 12px |
| TileSelector와 조합 | TileSelector 내부에 Radio 배치 | TileSelector padding 활용 |

- 수직 목록: 옵션이 3~5개일 때 가장 자연스러운 패턴
- 수평 배치: 옵션이 2~3개이고 텍스트가 짧을 때만 사용
- 카드 내부: 라벨에 부가 정보(가격, 소요 시간) 포함 가능

### Do / Don't
| Do | Don't | 이유 |
|----|-------|------|
| 같은 `name` 속성을 공유하는 Radio 그룹으로 묶기 | `name` 없이 Radio를 개별 배치 | `name` 없으면 브라우저가 그룹을 인식하지 못해 다중 선택 가능 |
| 반드시 각 Radio 옆에 라벨 텍스트 배치 | Radio만 단독으로 라벨 없이 배치 | 접근성 위반이며, 무엇을 선택하는지 알 수 없음 |
| 기본 선택(default checked) 항목을 제공 | 아무것도 선택되지 않은 상태로 초기화 | 사용자가 "무엇을 선택해야 하지?" 혼란 |
| 옵션 2~5개에서 사용 | 옵션 6개 이상에서 Radio 나열 | 화면 공간을 과도하게 차지. Dropdown이 효율적 |
| 밀집 UI에서는 `sm`, 모바일에서는 `lg` 사용 | 밀집된 테이블 행에서 `lg` 사이즈 사용 | 큰 라디오는 행 높이를 불필요하게 늘림 |
| `brand`는 주요 플로우, `black`은 부가 설정에 일관 적용 | 같은 폼 내에서 색상을 무작위로 섞기 | 색상이 선택의 중요도를 전달하므로 일관성 필요 |
