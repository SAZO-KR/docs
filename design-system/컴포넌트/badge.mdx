## 언제 사용하는가
정적 메타 정보(상태, 카테고리, 개수)를 시각적으로 강조하여 표시할 때 사용한다. Badge는 인터랙션이 없는 **정적 표시 전용** 컴포넌트.

## 대안 분기표
| 상황 | 사용할 컴포넌트 | 이유 |
|------|--------------|------|
| 클릭하여 선택/해제가 필요한 태그 | Chip | Badge는 정적 표시 전용, Chip은 토글형 인터랙티브 |
| 긴 안내 메시지 전달 | InfoPanel | Badge는 1~3단어 짧은 레이블 전용 |
| 액션 트리거 | Button | Badge는 `<span>`이므로 클릭 핸들링 부적합 |
| 숫자 카운트 (아이콘 위 알림 뱃지) | Notification Badge | Badge는 인라인 레이블, 오버레이가 아님 |

## 선택 가이드
## 색상 선택
색상은 Badge가 전달하는 의미(semantic)에 따라 결정한다. 같은 화면 내 일관성 유지 필수.

| Color | 의미 | 사용 맥락 | 예시 텍스트 |
|-------|------|----------|-----------|
| `default` | 중립, 일반 분류 | 카테고리, 속성 태그 | "의류", "무료배송" |
| `brand` | 브랜드 강조, 활성 | 현재 활성 상태, 신규 | "진행중", "NEW" |
| `purple` | 프로모션, 혜택 | 쿠폰, 할인, 이벤트 | "10% 할인", "TIP" |
| `error` | 오류, 위험 | 에러 상태, 품절 | "품절", "실패" |
| `warning` | 주의, 경고 | 재고 부족, 기한 임박 | "마감 임박", "주의" |
| `success` | 성공, 완료 | 완료 상태, 검증 통과 | "배송완료", "인증됨" |

## 크기 선택
| Size | 사용 맥락 | 배치 위치 |
|------|----------|---------|
| `xs` | 카드 내 밀집 영역, 작은 태그 | ItemCard 이미지 위, 테이블 셀 |
| `sm` | 기본 인라인 배지 (default) | 리스트 항목, 카드 상세 영역 |
| `md` | 강조 필요한 배지 | 상품 상세 페이지, 주문 상태 |
| `lg` | 독립적으로 눈에 띄어야 하는 배지 | 헤더, 프로모션 영역, 배너 |

## 형태 선택
| Rounding | 시각적 인상 | 사용 맥락 |
|----------|-----------|----------|
| `rounded` (pill) | 부드럽고 독립적 | 상태 표시, 개수, 인라인 레이블 |
| `square` | 정돈되고 구조적 | 카테고리 분류, 다른 UI와 정렬 |

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 텍스트 옆 인라인 | 제목/라벨 우측에 상태 표시 | 텍스트와 gap 4~8px |
| 카드 내부 | ItemCard 또는 커스텀 카드 내 메타 영역 | 다른 메타 정보와 gap 4px |
| 리스트 항목 | 리스트 행의 우측 또는 제목 아래 | 항목과 수평/수직 정렬 |
| 그룹 배치 | 여러 Badge 나란히 표시 (태그 목록) | gap 4px, flex-wrap 허용 |

## Do / Don't
| Do | Don't | 이유 |
|----|-------|------|
| 짧은 텍스트(1~3단어)로 상태 전달 | 긴 문장을 Badge에 넣음 | 한눈에 읽히는 레이블이어야 함 |
| 색상을 의미에 일관되게 사용 | 같은 의미에 다른 색상 혼용 | 색상-의미 매핑 혼란 유발 |
| `leadingAddon`에 아이콘으로 의미 강화 | addon에 인터랙티브 요소(버튼) 배치 | Badge는 정적 컴포넌트 |
| 한 화면 내 Badge 색상 최대 3가지 | 5가지 이상 색상 혼용 | 색상 구분력 저하 |
| `default`는 중립 분류/속성에만 사용 | `default`를 에러/성공 피드백에 사용 | 회색은 시각적 긴급성 전달 불가 |
| 목록/카드 내 일관된 위치에 배치 | 같은 목록 내 위치가 항목마다 다름 | 시선 예측 일관성 유지 |
