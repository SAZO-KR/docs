## 언제 사용하는가
여러 옵션 중 하나 이상을 선택/해제하는 토글형 인터랙션에 사용한다. 필터, 카테고리, 태그 등의 선택 상태를 시각적으로 표현.

## 대안 분기표
| 상황 | 사용할 컴포넌트 | 이유 |
|------|--------------|------|
| 정적 상태/카테고리 표시 (클릭 불가) | Badge | Chip은 토글형, 정적 표시에는 Badge |
| 상호 배타적 단일 선택 | Radio 또는 TileSelector | "하나만 선택" 의미가 명확한 컴포넌트 |
| 액션 트리거 (제출, 삭제) | Button | Chip은 상태 토글, 액션 실행 아님 |
| 넓은 영역 옵션 카드 | TileSelector | 이미지+텍스트+가격 등 복합 콘텐츠 |
| 페이지 이동 내비게이션 | Tabs 또는 Link | Chip 선택이 URL 변경을 유발하면 안 됨 |

## 선택 가이드

<Tabs>

<Tab title="색상 선택">

| Color | 시각적 특성 | 사용 맥락 | 선택 시 스타일 |
|-------|-----------|----------|-------------|
| `default` | 흰 배경 + 명확한 테두리 | 카테고리 필터, 옵션 선택 | 반전 (어두운 배경 + 흰 텍스트) |
| `gray` | 연회색 배경 + 연한 테두리 | 태그 표시, 보조 필터 | 반전 (어두운 배경 + 흰 텍스트) |
| `filter` | 연회색 배경 + 테두리 없음 | 검색 필터, 정렬 옵션 | 파란 배경 + 파란 텍스트 |

</Tab>

<Tab title="크기 선택">

| Size | 근사 높이 | 사용 맥락 | 배치 위치 |
|------|---------|----------|---------|
| `md` | 32px | 기본 칩, 데스크톱 필터 | 필터 바, 카테고리 영역 |
| `lg` | 40px | 모바일 터치 최적화 | 모바일 상단 필터, 전체 너비 옵션 |

</Tab>

<Tab title="배치 규칙">

| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 수평 스크롤 | Swiper 또는 overflow-x-auto로 수평 배치 | 칩 간 gap 4~8px |
| 줄바꿈 (flex-wrap) | 모든 칩을 화면 내 표시 | 칩 간 gap 8px, 행 간 gap 8px |
| 필터 바 상단 고정 | 검색 결과 위에 고정 | 컨텐츠와 gap 12~16px |
| 섹션 내부 | 카드/상세 페이지 내 옵션 선택 | 섹션 padding 내부 |

</Tab>

</Tabs>

## Do / Don't

<Check>

**Do**

- 간결한 텍스트(1~3단어) — *`whitespace-nowrap`으로 한 줄 표시*
- 복수 선택 가능한 필터/옵션에 사용 — *단일 선택은 Radio/TileSelector*
- `filter`는 검색/필터 전용으로 일관 사용 — *파란 선택 스타일이 "필터 활성" 의미*
- 칩 선택은 같은 페이지 내 콘텐츠 필터링 — *칩은 현재 뷰의 필터*
- 그룹 내 크기와 색상 통일 — *그룹 내 일관성 유지*
- 선택 상태 변화 즉각 반영 — *토글 피드백은 즉시 제공*

</Check>

<Warning>

**Don't**

- 긴 문장이나 설명
- 단일 선택 상황에서 사용
- `filter`를 비필터 맥락(프로필 태그)에 사용
- 칩 선택이 페이지 이동(라우팅) 유발
- 같은 그룹 내 크기/색상 혼합
- 선택 후 반응 없이 비동기 처리만

</Warning>
