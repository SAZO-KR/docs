## 언제 사용하는가
옵션 중 하나 또는 여러 개를 **넓은 터치 영역의 카드 형태** 로 선택할 때 사용한다. 내부 콘텐츠에 제약이 없어 복합 레이아웃이 가능하다. 체크박스/라디오의 **선택/해제 상태 전환** 이 본질이다.

사용 조건:
- 한 섹션 내에서 **3개 이상 옵션** 을 바닥에 깔아놓고 보여주고 싶을 때
- 복잡한 요소를 클릭 가능하게 만들되 라디오/체크박스로 동작해야 할 때 — **시각적 그루핑**
- 공간 여유가 있을 때 **터치 타겟 확대** 의도

## 대안 분기표
| 상황 | TileSelector 대신 사용할 컴포넌트 | 이유 |
|------|-------------------------------|------|
| 단순 텍스트 단일 선택 (옵션 2~5개) | Radio | 텍스트만 있으면 Radio가 공간 효율적 |
| 다중 선택 체크 (약관, 설정) | Checkbox | 체크 표시가 의미적으로 더 적합 |
| 필터/태그 토글 | Chip | 필터 맥락에서는 Chip이 가볍고 효율적 |
| 페이지 이동 네비게이션 | TileButton 또는 Link | TileSelector는 선택 컨트롤이지, 네비게이션이 아님 |
| 순수 정보 표시 (선택 불가) | 일반 카드 또는 InfoPanel | TileSelector는 선택 가능한 인터랙티브 요소 |

<Note>

**TileSelector vs TileButton** : TileSelector는 **상태 선택** (selected/deselected), TileButton은 **액션 트리거** (클릭 → 이동/실행). 선택인지 액션인지로 판단한다.

**TileSelector vs 투명 배경 리스트** : 모달/팝오버 안의 단순 메뉴 선택 → 투명 배경 리스트 컴포넌트. 페이지 내 다른 요소 근처에 배치하여 시각적 구분이 필요 → TileSelector.

</Note>

| `lg` | 32px (all) | 대형 옵션 카드, 강조 선택 | 플랜 선택, 이미지+텍스트+가격 조합 |

<Tab title="내부 콘텐츠 구성">

- Default: `bg-surface-default` + `border-border-default` + `shadow-lowest` (`--shadow-lowest`)

- Disabled: `bg-surface-disabled` -- cursor: not-allowed

</Tab>

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 2열 그리드 | 옵션 2~4개를 2열로 배치 | 카드 간 gap 8~12px |
| 3열 그리드 | 옵션이 많고 짧은 라벨일 때 | 카드 간 gap 8px |
| 단일 열 풀 너비 | 옵션별 상세 정보가 많을 때 | 행 간 gap 8~12px |
| 수평 스크롤 | 옵션이 화면 너비를 초과할 때 | 카드 간 gap 8px |

TileSelector는 `width: 100%`, `height: 100%` 기본 스타일을 가지므로 그리드 셀을 자동으로 채운다.

## Do / Don't

<Check>

**Do**

- children에 복합 레이아웃을 자유롭게 구성 (이미지+텍스트+가격) — *무엇을 선택하는지 명확해야 함. 라벨 텍스트는 필수*
- 그리드 레이아웃으로 배치하여 옵션을 한눈에 비교 가능하게 — *카드 형태의 강점은 시각적 비교*
- 선택 상태의 시각적 차이가 명확하도록 주변 여백 확보 — *선택 시 2px 테두리가 인접 카드와 겹치면 구분 불가*
- disabled 옵션은 전체의 1/3 이하로 제한 — *대부분 비활성이면 UX 혼란*
- 같은 그리드 내 모든 TileSelector의 크기(size)를 통일 — *카드 높이가 들쭉날쭉하면 그리드 정렬이 깨짐*
- 선택 불가한 이유가 있으면 disabled 카드에 설명 표시 — *사용자가 왜 선택할 수 없는지 알아야 대안을 찾을 수 있음*

</Check>

<Warning>

**Don't**

- 내부를 비워두거나 아이콘만 배치
- 옵션을 세로로 길게 나열
- 카드 간 gap 없이 밀착 배치
- disabled 옵션이 과반일 때 TileSelector 사용
- 같은 그리드 내에서 크기를 혼합
- disabled 상태를 이유 없이 방치

</Warning>
