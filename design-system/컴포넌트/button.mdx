## 언제 사용하는가
사용자가 명시적으로 액션을 트리거해야 하는 모든 곳에 사용한다. 제출, 확인, 취소, 탐색 등의 범용 액션 컴포넌트.

## 대안 분기표
| 상황 | 사용할 컴포넌트 | 이유 |
|------|--------------|------|
| 필터/카테고리 선택 | Chip | 토글형 선택이며, 액션 트리거가 아님 |
| 켜기/끄기 즉시 반영 설정 | Toggle | 이진 상태 전환이며, 제출이 아님 |
| 페이지 이동 (내비게이션) | Link | 시맨틱하게 `<a>` 요소가 적합 |
| 넓은 터치 영역이 필요한 옵션 카드 선택 | TileSelector | 복합 콘텐츠가 필요한 선택 UI |

## 선택 가이드
### 변형 선택
| Variant | Border Style | 시각적 인상 | 사용 맥락 |
|---------|-------------|-----------|----------|
| outline (기본) | solid 1px | 배경 투명, 가벼움 | 보조 액션, 취소, 부차적 CTA |
| `filled` | solid 1px | 배경 채움, 시각적 무게 | 주요 CTA, 확인, 제출 |
| `dash` | dashed 1px | 점선 테두리, "미완성/선택적" | 항목 추가, 선택사항 |

## 색상 선택
| Color | 의미 | 사용 맥락 | 예시 |
|-------|------|----------|------|
| (none) | 기본/중립 | 범용 보조 액션 | 취소, 닫기 |
| `brand` | 핵심 행동 유도 | 화면의 메인 CTA | 결제하기, 장바구니 담기 |
| `error` | 파괴적/위험 | 되돌리기 어려운 행동 | 삭제하기, 탈퇴하기 |
| `purple` | 혜택/프로모션 | 쿠폰, 할인 관련 | 쿠폰 적용, 할인 받기 |
| `gray` | 중립 강조 | outline에서 텍스트+테두리 강조 | 뒤로 가기, 나중에 |

## 크기 선택
| Size | 근사 높이 | 사용 맥락 | 배치 위치 |
|------|---------|----------|---------|
| `xl` | 56px | 메인 CTA (결제, 로그인, 가입) | 화면 하단 고정, 풀 너비 |
| `lg` | 48px | 페이지 레벨 — 단일 버튼 또는 단일 옵션 그룹 (1~2개) | 섹션 내부, 풀 너비 또는 컨테이너 너비 |
| `md` | 40px | 섹션 레벨 — 섹션별 1번 사용하는 단일/그룹 버튼 | 카드 내부, 모달 Footer, 폼 액션 |
| `sm` | 36px | 하위 섹션 레벨 — 한 뷰포트 내 2번 이상 반복 가능한 버튼 | 리스트 항목 우측, 테이블 행, 인라인 |

크기 선택 핵심 원칙: **반복 빈도가 높을수록 작은 버튼** . 페이지에 1번 등장하면 lg, 섹션에 1번이면 md, 여러 번 반복되면 sm.

## 버튼 계층
한 화면에 여러 버튼이 있을 때, variant와 color 조합으로 시각적 계층을 만든다.

| 계층 | Variant | Color | 사용 맥락 | 예시 |
|------|---------|-------|----------|------|
| Primary Action | `filled` | `brand` | 화면의 핵심 행동 -- 화면당 1개 | 결제하기, 회원가입 |
| Secondary Action | outline | (none) / `gray` | Primary 옆 또는 아래 | 취소, 뒤로 가기 |
| Tertiary Action | `dash` | (none) / `gray` | 선택적/보조 액션 | 주소 추가, 쿠폰 입력 |
| Destructive Action | `filled` / outline | `error` | 파괴적 행동 | 삭제하기, 탈퇴하기 |
| Promotional Action | `filled` / outline | `purple` | 프로모션 관련 | 쿠폰 적용 |

## 배치 규칙
| 배치 패턴 | 구성 | 간격 |
|----------|------|------|
| 하단 고정 CTA | `xl` filled brand, 풀 너비 | 좌우 패딩 20px |
| 바텀 버튼 컨테이너 | 페이지 내 최중요 액션 배치. **모바일 전용** — PC에는 이 개념이 없음 | - |
| 모달 Footer (DoubleButton) | `lg` outline + `lg` filled | gap 8px |
| 모달 Footer (Single) | `lg` filled brand, 풀 너비 | - |
| 인라인 버튼 그룹 | `md` 또는 `sm` 여러 개 | gap 8px, 주요 버튼 우측 |
| 카드 내부 | `sm` 또는 `md` | 카드 하단 패딩 내 |

- 버튼이 여러 개이고 텍스트가 길 경우 버튼을 **수직 배치** 한다.

## Do / Don't

<Check>

**Do**

- 구체적 행동 라벨 사용 ("결제하기", "장바구니 담기") — *라벨만으로 클릭 결과를 예측 가능해야 함*
- 파괴적 액션에 `error` 색상 사용 — *위험을 시각적으로 즉시 인지시켜야 함*
- `filled` + `outline` 조합으로 계층 구분 — *시각적 중요도 차이가 명확해야 함*
- 로딩 중 스피너 + `disabled` 처리 — *중복 제출 방지 + 처리 중 안심*
- 아이콘은 `lead`/`tail` 중 하나만 — *라벨 가독성 저하*
- disabled 시 이유 안내 (tooltip, 문구) — *활성화 조건을 알 수 없음*
- `dash`는 보조 액션 전용 — *점선은 "선택적/미완성" 인상*
- `filled` 버튼은 화면당 최대 1~2개 — *시각적 위계 붕괴*

</Check>

<Warning>

**Don't**

- 모호한 라벨 ("확인", "OK")
- 파괴적 액션에 `brand` 색상
- `filled` + `filled` 조합
- 클릭 후 아무 피드백 없이 방치
- 양쪽에 동시 아이콘 배치
- disabled 상태 이유 없이 방치
- `dash`를 메인 CTA로 사용
- `filled` 3개 이상 나열

</Warning>
