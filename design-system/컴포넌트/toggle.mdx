## 언제 사용하는가
이진 상태(켜짐/꺼짐)를 전환하며, 변경이 **즉시 반영** 되는 설정에 사용한다.

## 대안 분기표
| 상황 | 사용할 컴포넌트 | 이유 |
|------|--------------|------|
| 폼 제출 후 반영되는 동의/선택 | Checkbox | Toggle은 즉시 반영, Checkbox는 "제출 후 반영" |
| 3개 이상 옵션 중 하나 선택 | Radio | Toggle은 이진(on/off) 전용 |
| 복수 항목 선택/해제 (필터) | Chip | 여러 옵션 토글은 Chip 그룹 |
| 명시적 확인이 필요한 약관 동의 | Checkbox | 약관 동의는 "읽고 동의" 행위 |

## 선택 가이드
### 색상 선택
| Color | On 상태 | 사용 맥락 | 예시 |
|-------|---------|----------|------|
| `brand` | 브랜드 레드 | 주요 설정, 핵심 기능 | 알림 수신, 마케팅 동의, 자동 결제 |
| `black` | 다크 그레이 | 부가 설정, 중립 옵션 | 다크모드, 상세 설정, 표시 옵션 |

### 크기 선택
| Size | Track 크기 | 사용 맥락 | 배치 위치 |
|------|-----------|----------|---------|
| `sm` | 36x20px | 밀집된 설정 목록, 테이블 | 설정 리스트 항목 우측 |
| `md` | 44x24px | 기본 토글, 독립 설정 | 설정 페이지, 프로필 |
| `lg` | 52x28px | 강조 설정, 모바일 핵심 | 모바일 설정 메인, 온보딩 |

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 설정 리스트 항목 | 라벨(좌) + Toggle(우) 수평 정렬 | `space-between` |
| 카드 내부 설정 | 카드 내 옵션 행 우측 | 행 간 gap 12~16px |
| 독립 설정 | 설명 텍스트 옆에 단독 배치 | 설명과 gap 8~12px |
| 폼 내부 | 지양 -- Checkbox가 적합 | Toggle은 즉시 반영이므로 폼 제출과 충돌 |

## Do / Don't

<Check>

**Do**

- 토글 옆에 현재 상태 설명 라벨 배치 — *무엇을 켜고 끄는지 알 수 없음*
- 상태 변경 즉시 반영 + 피드백 (토스트 등) — *설정 반영 확신 불가*
- 밀집 설정 목록에서 `sm` 사용 — *행 높이 불필요하게 증가*
- 즉시 반영 설정에 Toggle 사용 — *"즉시 반영" 기대 깨짐*
- disabled 시 비활성 이유 보조 텍스트 — *사용자 좌절 유발*
- `brand`=핵심, `black`=부가로 일관 구분 — *색상이 중요도를 전달해야 함*

</Check>

<Warning>

**Don't**

- 토글만 단독으로 설명 없이 배치
- 토글 전환 후 아무 피드백 없이 서버 요청만
- 밀집 리스트에서 `lg` 사용
- 폼 제출 필요 설정에 Toggle 사용
- disabled 설명 없이 방치
- 같은 설정 페이지 내 색상 무작위 혼용

</Warning>
