## 언제 사용하는가
사용자가 키보드로 자유 형식 텍스트를 입력해야 하는 모든 곳에 사용한다. 라벨, 힌트, 선행/후행 요소를 포함한 완결된 단일 행 입력 UI.

## 대안 분기표
| 상황 | 사용할 컴포넌트 | 이유 |
|------|--------------|------|
| 미리 정해진 옵션 중 선택 | Radio, Chip, TileSelector | 자유 입력이 아닌 선택형 UI가 적합 |
| 긴 여러 줄 텍스트 입력 | Textarea | 단일 행 Input은 한 줄 텍스트 전용 |
| 켜기/끄기 이진 선택 | Toggle, Checkbox | 텍스트 입력 불필요 |
| 날짜/시간 선택 | DatePicker | 구조화된 입력에는 전용 UI가 적합 |

## 선택 가이드
### 슬롯 구성 기준
| 슬롯 | 배치 요소 | 사용 맥락 | 예시 |
|------|---------|----------|------|
| `label` | 필드 목적 설명 | 모든 Input에 필수 | "이메일 주소", "검색어" |
| `extraLabel` | 보조 정보 | 입력 제한, 옵션 표시 | "0/50", "(선택)" |
| `lead` | 정적 아이콘 | 필드 유형 즉시 인지 | 검색 돋보기, 이메일 봉투 |
| `tail` | 인터랙티브 요소 | 보조 액션 | 입력 지우기(X), 비밀번호 보기/숨기기 |
| `hint` | 도움말 또는 에러 메시지 | 형식 안내, 유효성 결과 | "영문+숫자 8자 이상" |

## 상태 전환 시나리오
| 시나리오 | color prop | hint 내용 | 시각적 피드백 |
|----------|-----------|----------|-------------|
| 초기 진입 (안내) | (default) | 입력 형식 가이드 | `border-border-default` 테두리, 기본 힌트 |
| 입력 중 (포커스) | (default) | 유지 또는 숨김 | `border-border-strong` 테두리 (강조) |
| 유효성 통과 | (default) | 성공 메시지 또는 숨김 | 기본 상태 복원 |
| 유효성 실패 | `error` | 구체적 에러 메시지 | `border-border-error` + `text-text-error` 라벨/힌트 + 연분홍 배경 |
| 비활성 | disabled | 비활성 이유 안내 | `bg-surface-disabled` 배경, 입력 불가 커서 |

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 수직 폼 (기본) | Input을 위아래로 쌓음 | 필드 간 gap 16~24px |
| 수평 인라인 | 짧은 필드 나란히 배치 (우편번호 + 상세주소) | gap 8~12px |
| 모달 내부 | 모달 content 영역 배치 | 모달 padding (20px) 활용 |
| 검색 바 | `lead`에 검색 아이콘, `tail`에 지우기 버튼 | 단독 사용, 풀 너비 |

## Do / Don't

<Check>

**Do**

- 항상 `label` 제공하여 필드 목적 명시 — *placeholder는 입력 시 사라짐*
- `color="error"` 시 `hint`에 구체적 에러 메시지 — *무엇이 잘못됐는지 알 수 없음*
- `id` 제공하여 label-input 접근성 연결 — *스크린 리더 인식 불가*
- `lead`에 정적 아이콘, `tail`에 인터랙티브 요소 — *lead는 시각적 힌트 전용*
- placeholder에 입력 예시 ("예: sazo@example.com") — *구체적 예시가 효과적*
- disabled 시 비활성 이유 안내 — *수정 불가한데 에러 표시하면 혼란*
- 관련 필드끼리 그룹핑 (전화번호 국번 + 번호) — *시각적 그룹핑 = 논리적 관계*
- `lead`와 `tail` 중 하나만 사용 — *입력 영역이 좁아짐*

</Check>

<Warning>

**Don't**

- `placeholder`를 label 대용으로 사용
- 에러 상태에서 hint 없이 빨간 테두리만
- `id` 없이 시각적으로만 연결
- `lead`에 클릭 가능한 버튼 배치
- "입력하세요" 같은 불필요한 안내
- disabled + error 동시 적용
- 관련 없는 필드를 같은 행에 배치
- 양쪽 모두 아이콘 배치

</Warning>
