## 언제 사용하는가
사용자에게 문맥적 안내, 경고, 피드백을 **배너 형태** 로 전달할 때 사용한다. 정적 표시 전용이며, `link` 슬롯으로 후속 행동을 유도할 수 있다.

## 대안 분기표
| 상황                  | InfoPanel 대신 사용할 컴포넌트 | 이유                         |
| ------------------- | --------------------- | -------------------------- |
| 짧은 인라인 상태 레이블       | Badge                 | 1~3단어 수준의 메타 정보는 Badge가 적합 |
| 즉각적인 피드백 (성공/실패 알림) | Toast (Snackbar)      | 일시적 피드백은 자동 사라지는 Toast가 적합 |
| 사용자 확인이 필요한 중요 결정   | SazoModal (Dialog)    | 반드시 읽고 행동해야 하는 상황은 모달로 차단  |
| 폼 필드별 에러 메시지        | Input의 `hint` 슬롯      | 필드 수준 에러는 Input에 직접 표시     |

## 선택 가이드
### 색상 선택
색상은 메시지의 **의미(semantic)** 에 따라 결정한다. 잘못된 색상 선택은 사용자에게 혼란을 줄 수 있다.

| Color | 의미 | 사용 맥락 | 예시 메시지 |
|-------|------|-----------|-----------|
| `default` | 일반 안내, 중립 정보 | 기능 설명, 도움말, 보충 설명 | "해외배송 상품은 반품이 불가합니다." |
| `error` | 오류, 실패, 위험 | 결제 실패, 입력 오류, 서비스 장애 | "결제에 실패했습니다. 다시 시도해주세요." |
| `warning` | 주의, 유의 사항 | 기한 임박, 재고 부족, 제한 안내 | "이 쿠폰은 내일까지만 사용 가능합니다." |
| `success` | 성공, 완료, 확인 | 주문 완료, 설정 저장, 인증 성공 | "주문이 성공적으로 접수되었습니다." |
| `blueDark` | 정보성 안내, 팁 | 사용 방법, 기능 소개, 정책 안내 | "일본 직구 상품은 관부가세가 별도 부과됩니다." |
| `purple` | 프로모션, 혜택 안내 | 쿠폰, 할인, 이벤트 | "첫 구매 시 5,000원 할인 쿠폰이 적용됩니다." |

- `default`는 `bg-surface-secondary` / `border-border-subtle` 조합
- `error`/`warning`/`success`는 시맨틱 토큰의 피드백 계층 사용
- `purple`은 `bg-surface-benefit` 배경 -- 사용자에게 긍정적 기회를 전달하는 SAZO 고유 패턴

### 크기 선택
| Size | Border Radius | 사용 맥락 | 배치 위치 |
|------|--------------|-----------|-----------|
| `xs` | - | **뱃지의 확장판** — 뱃지보다 긴 디스크립션이 필요할 때 | 인라인, 컴포넌트 근접 |
| `sm` | `rounded-xl` | 섹션 내부 안내, 카드 내 인라인 알림 | 폼 상단, 카드 내부, 섹션 도입부 |
| `md` | `rounded-2xl` | 독립적 안내 영역, 주요 알림 | 페이지 컨텐츠 영역, 모달 내부 |
| `md-full` (모바일) / `md` (PC) | - | **페이지 전체** 에 해당하는 안내 | 페이지 상단 또는 컨텐츠 영역 |
| `full` | `rounded-none` | 페이지 전폭 배너, 상단 공지 | 페이지 최상단, 모바일 전폭 배너 |

### 슬롯 구성
| 슬롯 | 배치할 요소 | 사용 맥락 |
|------|-----------|-----------|
| `lead` | FeaturedIcon (색상에 맞는 아이콘) | 메시지 유형을 시각적으로 즉시 인지시킬 때 |
| `content` (children) | 안내 텍스트 (2~3줄 이내) | 핵심 정보 전달 |
| `tail` | 닫기 버튼 (X 아이콘) | 사용자가 패널을 닫을 수 있을 때 |
| `link` | CTA 링크 ("자세히 보기", "쿠폰 받기") | 사용자에게 후속 행동을 유도할 때 |

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 폼 상단 안내 | 폼 필드 위에 `sm` 크기로 배치 | 폼과 gap 16px |
| 페이지 상단 배너 | `full` 크기로 페이지 최상단 배치 | 컨텐츠와 gap 0 (연속) |
| 섹션 도입부 | 섹션 제목 아래, 콘텐츠 위에 배치 | 위아래 gap 12~16px |
| 모달 내부 | 모달 content 영역 상단에 배치 | 모달 padding 활용 |

## Do / Don't
| Do | Don't | 이유 |
|----|-------|------|
| 색상을 메시지 의미에 맞게 선택 (`error`=오류, `warning`=주의) | `default` 색상을 위험한 상황에 사용 | 회색은 긴급성을 전달하지 못해 사용자가 중요 정보를 놓침 |
| 짧고 명확한 문장으로 핵심 정보 전달 (2~3줄 이내) | 장문의 텍스트를 InfoPanel에 넣음 | 배너 내 텍스트가 길면 읽히지 않고 무시됨 |
| `lead`에 FeaturedIcon을 배치하여 메시지 유형을 시각적으로 강화 | `lead` 없이 텍스트만으로 메시지 유형 전달 | 아이콘이 있으면 색상과 함께 메시지 유형을 즉각 인지 |
| 사용자 행동이 필요하면 `link` 슬롯에 CTA 배치 | `link` 없이 사용자에게 행동을 요구하는 텍스트만 표시 | 행동 수단이 없으면 답답함 |
| 한 화면에 InfoPanel은 최대 2개, 색상은 서로 다르게 | 같은 색상의 InfoPanel을 여러 개 수직 나열 | 정보 과부하로 어떤 패널이 중요한지 구분 불가 |
| `full` 크기는 페이지 전폭 배너에만 사용 | `full` 크기를 카드 내부나 좁은 섹션에 사용 | 전폭 배너가 좁은 영역에 있으면 시각적으로 어색 |
