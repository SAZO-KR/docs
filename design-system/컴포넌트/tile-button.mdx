## 언제 사용하는가
카드 형태의 넓은 터치 영역으로 **색상 코딩된 피드백 정보와 함께 사용자의 주의를 유도** 하는 액션 버튼. 주문 상태, 배송 추적 등 상태 정보 카드로 주로 활용된다.

일반 버튼과 다른 층위의 버튼이 필요할 때 사용하며, 두 조건이 **동시에** 성립해야 한다:
1. 일반 버튼 변형(브랜드/짙은 그레이/연한 그레이/흰색)을 **모두 소진** 했거나, 화면/섹션의 의도상 낮은 계층의 버튼만 사용해야 하는데 그것도 이미 사용한 상황 — 즉, 버튼 위계 내에서 더 이상 차별화할 수단이 없을 때
2. 텍스트가 길어서 일반 버튼에 담으면 공간이 부족하거나, 담았을 때 버튼이 텍스트로 가득 차 시각적으로 답답해 보일 때

## 대안 분기표
| 상황 | TileButton 대신 사용할 컴포넌트 | 이유 |
|------|---------------------------|------|
| 일반적인 CTA (확인, 제출, 저장) | Button | TileButton은 정보 카드형이지, 폼 제출 버튼이 아님 |
| 옵션 선택 (배송 방법, 크기) | TileSelector | TileButton은 액션 트리거, TileSelector는 상태 선택 |
| 안내/경고 배너 (클릭 불가) | InfoPanel | 인터랙션 없는 정보 표시는 InfoPanel |
| 네비게이션 링크 (텍스트만) | Link | 텍스트만 있으면 Link가 가볍고 적합 |
| 짧은 인라인 상태 레이블 | Badge | 1~3단어 메타 정보는 Badge |

<Info>

**TileButton vs TileSelector** : TileButton은 **액션 트리거** (클릭 → 이동/실행), TileSelector는 **상태 선택** (selected/deselected). 클릭 후 무엇이 일어나는지로 판단한다.

</Info>

## 선택 가이드

<Tabs>

<Tab title="색상 선택">

색상은 정보의 **성격(semantic)** 을 전달한다. 장식용으로 사용하지 않는다.

| Color | 의미 | 사용 맥락 | 토큰 |
|-------|------|-----------|------|
| (default) | 중립 | 일반 네비게이션, 정보 카드 | `bg-surface-default` (`--surface-default`) |
| `brand` | 브랜드 강조 CTA | 주요 액션, 핵심 상태 | rose-50 배경 (아래 주석 참조) |
| `error` | 오류/위험 | 주의 필요 액션, 결제 오류 안내 | `bg-surface-error` (`--surface-error`) |
| `warning` | 경고/주의 | 기한 임박, 재고 부족 | `bg-surface-warning` (`--surface-warning`) |
| `success` | 성공/완료 | 배송 완료, 처리 성공 | `bg-surface-success` (`--surface-success`) |
| `purple` | 프로모션/혜택 | 쿠폰, 할인, 이벤트 | `bg-surface-benefit` (`--surface-benefit`) |
| `gray` | 보조 정보 | 부가 설정, 보조 네비게이션 | `bg-surface-secondary` (`--surface-secondary`) |

<Info>

**NOTE (brand = rose 팔레트)** : `brand` color는 현재 brand-500이 아닌 **rose 팔레트** (rose-50/500/600/700)를 사용한다. 이는 구현상의 특이점이며, 향후 brand 팔레트로 마이그레이션이 예정되어 있다.

</Info>

</Tab>

<Tab title="슬롯 구성">

| 슬롯 | 배치할 요소 | 사용 맥락 |
|------|-----------|-----------|
| `lead` | 상태 아이콘 (FeaturedIcon 등) | 시각적 인지 보강 |
| `children` | 상태 텍스트 (1~2줄) | 핵심 정보 전달 |
| `tail` | ChevronRight 아이콘 | 네비게이션 가능함을 암시. **Chevron 아이콘이 핵심 어포던스** — "클릭하면 뭔가 진행된다, 특히 이동할 것 같다"는 행동 유도성을 만들어냄 |

</Tab>

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 수직 리스트 | 상태 카드를 위에서 아래로 나열 | 항목 간 gap 8~12px |
| 2열 그리드 | 동일 카테고리의 상태 카드 비교 | 카드 간 gap 8~12px |
| 단일 카드 | 독립적인 CTA 또는 상태 안내 | 주변 컨텐츠와 gap 16px |

- `tail`에 ChevronRight를 넣으면 "이동 가능" 시그널을 명확히 전달한다.
- `lead`에 상태 아이콘을 배치하면 색상과 함께 의미를 이중으로 전달한다.

## Do / Don't

<Check>

**Do**

- 색상을 통해 정보의 성격(성공/경고/에러)을 직관적으로 전달 — *색상 코딩이 무의미해짐*
- `tail`에 ChevronRight 아이콘을 넣어 네비게이션 가능함을 암시 — *클릭 가능성 시그널 부재*
- `lead`에 상태 아이콘을 배치하여 시각적 인지를 도움 — *아이콘이 없으면 텍스트만으로 의미 전달이 약함*
- 1~2줄의 간결한 텍스트를 children에 사용 — *카드형 버튼은 긴 텍스트에 적합하지 않음*
- 한 화면에서 색상 다양성을 3~4가지 이내로 제한 — *색상 코딩이 무의미해짐*
- 일반적인 CTA(확인, 제출)에는 Button 사용 — *TileButton은 정보 카드형 액션이지, 폼 제출이 아님*
- 항상 누를 수 있는 상황에서만 렌더링 — *TileButton에는 Disabled가 없다 — 핵심 CTA가 아니라 보조 동작(이동, 부가 액션)을 수행하므로 조건부 비활성화가 발생하지 않음*

</Check>

<Warning>

**Don't**

- color의 의미를 무시하고 단순 장식용으로 사용
- tail 없이 네비게이션을 기대하게 하는 텍스트 사용
- lead/tail 모두 비워두기
- 긴 본문 텍스트를 children에 넣기
- 서로 다른 색상의 TileButton을 과도하게 혼용
- 폼 제출 버튼으로 TileButton 사용
- Disabled 상태를 구현

</Warning>
