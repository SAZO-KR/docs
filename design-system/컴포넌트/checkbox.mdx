## 언제 사용하는가
여러 항목 중 독립적으로 선택/해제할 수 있는 다중 선택 컨트롤. 주로 **폼 제출 전 확인이 필요한 맥락** 에서 사용한다.

## 대안 분기표
| 상황 | 사용할 컴포넌트 | 이유 |
|------|--------------|------|
| 상호 배타적 단일 선택 | Radio | Checkbox는 다중 선택 전용 |
| 즉시 반영되는 이진 설정 (on/off) | Toggle | Checkbox는 "제출 후 반영", Toggle은 "즉시 반영" |
| 필터 태그 선택/해제 | Chip | 필터 맥락에서 시각적으로 가벼운 Chip이 적합 |
| 넓은 터치 영역 필요한 옵션 카드 | TileSelector | 복합 콘텐츠가 포함된 선택 |

## 선택 가이드

<Tabs>

<Tab title="변형 선택">

| Variant | Checked 시 스타일 | 사용 맥락 | 예시 |
|---------|----------------|----------|------|
| `filled` | 배경 채움 + 흰색 체크 아이콘 | 명확한 확인 필요, 시각적 강조 | 약관 동의, 전체 선택, 필수 체크 |
| `outline` | 투명 배경 + 색상 체크 아이콘 | 가벼운 선택, 밀집 목록 | 필터 옵션, 관심사 선택, 부가 설정 |

</Tab>

<Tab title="색상 선택">

| Color | Checked 시 색상 | 사용 맥락 | 예시 |
|-------|---------------|----------|------|
| `brand` | 브랜드 레드 | 주요 플로우 (주문, 가입, 결제) | 필수 약관, 전체 선택, 구매 옵션 |
| `black` | 다크 그레이 | 부가 설정, 관리 화면 | 알림 설정, 필터, 관리자 체크 |

</Tab>

<Tab title="크기 선택">

| Size | 크기 | 사용 맥락 | 배치 위치 |
|------|------|----------|---------|
| `sm` | 16x16px | 밀집 리스트, 테이블 행 | 테이블 셀, 좁은 목록 |
| `md` | 20x20px | 기본 체크박스, 폼 내부 | 약관 동의, 옵션 목록, 필터 |
| `lg` | 24x24px | 강조 선택, 모바일 터치 | 모바일 약관 동의, 핵심 체크 |

</Tab>

</Tabs>

## 배치 규칙
| 배치 패턴 | 설명 | 간격 |
|----------|------|------|
| 라벨과 수평 배치 | Checkbox(좌) + 라벨 텍스트(우) | gap 8px |
| 수직 옵션 목록 | 체크박스+라벨 행을 위아래 나열 | 행 간 gap 12~16px |
| 전체 선택 + 개별 선택 | 상단 "전체 선택" + 아래 개별 항목 | 구분선 + gap 12px |
| 테이블 내부 | 행 첫 번째 셀에 `sm` 체크박스 | 테이블 셀 padding 활용 |

## Do / Don't

<Check>

**Do**

- 반드시 옆에 라벨 텍스트 배치 — *접근성 위반, 선택 대상 불명*
- 약관 동의 등 명확한 확인에 `filled` 사용 — *`filled`가 "확실히 선택됨" 시각 강화*
- `brand`=주요 플로우, `black`=부가로 일관 — *색상이 중요도를 전달*
- 폼 제출 후 처리되는 설정에 Checkbox 사용 — *"체크해도 왜 안 바뀌지?" 혼란*
- disabled 시 비활성 이유 보조 텍스트 — *비활성 이유 모르면 좌절*
- "전체 선택"은 개별 항목과 시각적 분리 — *전체 선택의 일괄 제어 역할 인지*

</Check>

<Warning>

**Don't**

- 라벨 없이 체크박스만 단독 배치
- 중요한 동의 항목에 `outline` 사용
- 같은 폼 내 색상 무작위 혼용
- 즉시 실행 설정에 Checkbox 사용
- disabled을 설명 없이 과도하게 사용
- 전체 선택과 개별을 동일 레벨 나열

</Warning>
