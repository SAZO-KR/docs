## 페이지 제목과 첫 섹션 사이 불필요한 여백 수정

디자인 시스템 문서 페이지에서 페이지 제목(H1)과 첫 번째 섹션 제목(H2) 사이에 과도한 여백이 표시되던 문제를 수정했습니다.

- **제목 간격 최적화** — 페이지 제목 바로 아래 첫 섹션 제목의 불필요한 상단 여백이 제거되어 콘텐츠가 더 자연스럽게 이어집니다
- **다양한 레이아웃 대응** — 페이지 헤더 구조에 관계없이 첫 섹션 제목의 간격이 일관되게 표시됩니다
</Update>

<Update label="2026-04-21" description="v1.0.13" tags={["버그 수정"]}>
## 페이지 로딩 시 레이아웃 깜빡임 수정

디자인 시스템 문서 41개 페이지에서 발생하던 로딩 시 레이아웃 깜빡임(layout shift) 현상을 수정했습니다. 스타일이 전역으로 통합 적용되어 페이지가 더 안정적으로 표시됩니다.

- **레이아웃 깜빡임 해소** — 페이지 로딩 시 제목 간격과 색상이 뒤늦게 적용되며 콘텐츠가 밀리던 현상이 사라져 더 부드러운 로딩 경험을 제공합니다
- **스니펫 스타일 적용 개선** — 재사용 스니펫에서 스타일이 누락되던 문제가 해결되어 모든 페이지에서 일관된 디자인이 표시됩니다
</Update>

## 테마 색상 체계 변경

문서 사이트의 테마 색상 배치가 업데이트되었습니다. 라이트 모드와 다크 모드 각각의 강조 색상이 재배치되어 모드별 가독성이 개선됩니다.

- **라이트 모드 강조 색상 변경** — 기본 강조 색상이 브랜드 레드에서 다크 톤(`#222222`)으로 변경되어 라이트 모드에서 텍스트와 UI 요소의 대비가 향상됩니다
- **다크 모드 브랜드 색상 적용** — 다크 모드의 강조 색상이 브랜드 레드(`#EB4666`)로 변경되어 어두운 배경에서 주요 요소가 더 뚜렷하게 강조됩니다
</Update>

## 2026년 4월 21일 주간 변경 로그

이번 주 SAZO 디자인 시스템의 주요 변경 사항을 요약합니다.

### 새 기능

- **SAZO 디자인 시스템 출시** — Foundations, Guidelines, Components, Patterns, Content 5개 영역, 총 40개 이상의 페이지로 구성된 디자인 시스템이 정식 공개되었습니다. 디자이너, 기획자, 개발자가 일관된 기준으로 협업할 수 있습니다. [디자인 시스템 개요](/design-system/index)
- **인터랙티브 컴포넌트 적용** — 탭(Tabs), 아코디언(Accordion), 노트(Note) 등 인터랙티브 요소가 전체 디자인 시스템 페이지에 적용되어 관련 옵션을 빠르게 비교하고 상세 정보를 접고 펼칠 수 있습니다

### 업데이트

- **Pretendard Variable 서체 적용** — 제목과 본문 서체가 Pretendard Variable로 통일되어 한글·영문 혼용 환경에서 가독성이 향상되었습니다
- **문서 탐색 개선** — 40개 디자인 시스템 페이지의 목차 구조가 정리되어 원하는 정보를 빠르게 찾을 수 있습니다
- **사이트 테마 개선** — Mint 테마 적용과 다크 모드 배경 색상 변경으로 전체적인 레이아웃과 가독성이 개선되었습니다
- **Do/Don't 가이드 시각화** — 기존 테이블 형식의 Do/Don't 가이드가 체크/경고 블록으로 변경되어 모범 사례와 주의사항을 한눈에 파악할 수 있습니다. [버튼 위계](/design-system/가이드/버튼-위계), [카드 조합](/design-system/레시피/카드-조합) 등 24개 섹션에 적용
- **토큰 아키텍처 플로차트 도입** — [간격](/design-system/원칙/간격), [모션](/design-system/원칙/모션), [타이포그래피](/design-system/원칙/타이포그래피) 페이지의 토큰 구조도가 플로차트로 변경되어 계층 간 관계를 더 직관적으로 파악할 수 있습니다
- **라이트 모드 강조 색상 통일** — 라이트 모드와 다크 모드의 강조 색상이 통일되어 모드 전환 시 더 자연스러운 경험을 제공합니다

### 버그 수정

- **인터랙티브 컴포넌트 렌더링 오류 수정** — 35개 페이지에서 Tabs, Accordion, Steps, Note 내부 콘텐츠가 코드 블록으로 잘못 표시되던 문제를 수정했습니다
- **버튼 위계 페이지 렌더링 복구** — [버튼 위계](/design-system/가이드/버튼-위계) 페이지에서 특수 문자로 인해 페이지가 표시되지 않던 문제를 수정했습니다
- **다이어그램 레이아웃 수정** — [버튼 위계](/design-system/가이드/버튼-위계), [텍스트 계층](/design-system/가이드/텍스트-계층), [카드 조합](/design-system/레시피/카드-조합), [린터 스펙](/design-system/콘텐츠/린터-스펙) 페이지에서 코드 블록이 프레임과 충돌하여 레이아웃이 깨지던 문제를 수정했습니다
</Update>

## 디자인 시스템 시각적 표현 및 레이아웃 수정

토큰 아키텍처 다이어그램이 인터랙티브 플로차트로 개선되고, 일부 페이지의 스타일 및 레이아웃 문제가 수정되었습니다.

- **제목 스타일 적용 수정** — 일부 환경에서 디자인 시스템 페이지의 제목(h2, h3) 색상과 간격 스타일이 적용되지 않던 문제를 수정했습니다

## 디자인 시스템 문서 가독성 개선

- **섹션 간격 및 색상 강조** — 주요 섹션 제목(h2)에 브랜드 색상이 적용되고, 섹션 간 간격이 넓어져 긴 문서에서도 각 섹션의 시작점을 빠르게 찾을 수 있습니다

## 디자인 시스템 문서 인터랙티브 컴포넌트 적용

디자인 시스템 문서 35개 페이지에 인터랙티브 컴포넌트를 적용하여 콘텐츠 탐색이 더 편리해졌습니다.

- **탭(Tabs) 도입** — 선택 가이드, 맥락별 적용, PC/Mobile 비교 등 관련 옵션을 탭으로 전환하여 한 화면에서 빠르게 비교할 수 있습니다
- **아코디언(Accordion) 도입** — 구조, 토큰, UI 패턴, 설계 기록 등 상세 정보를 접고 펼 수 있어 페이지가 더 깔끔해졌습니다
- **노트(Note) 강조** — 주의사항과 참고 정보가 시각적으로 구분되어 중요한 안내를 놓치지 않게 됩니다
- 전체적으로 긴 문서의 가독성이 개선되어 원하는 정보를 더 빠르게 찾을 수 있습니다
</Update>

## 서체 변경: Pretendard Variable 적용

문서 사이트 전체의 서체를 **Pretendard Variable**로 변경했습니다. 제목과 본문 모두 동일한 서체가 적용되어 더욱 일관된 한국어 가독성을 제공합니다.

- 제목(heading)과 본문(body) 서체가 Pretendard Variable로 통일되었습니다

## SAZO 디자인 시스템 최초 공개

SAZO 디자인 시스템 문서가 정식 출시되었습니다. 디자이너, 기획자, 개발자 모두가 일관된 기준으로 협업할 수 있도록 체계적인 가이드를 제공합니다.

### Foundations

디자인 시스템의 기반이 되는 토큰과 원칙을 문서화했습니다.

- **[디자인 원칙](/design-system/원칙/디자인-원칙)** — 정직한 안내자, 본질만 남긴 간결함, 체계적 확장의 3대 설계 원칙

- **[타이포그래피](/design-system/원칙/타이포그래피)** — 타이틀, 본문, 라벨 등 역할 기반 타이포그래피 스케일
